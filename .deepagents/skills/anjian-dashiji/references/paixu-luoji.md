# 排序逻辑法

按时间顺序组织事件，形成清晰的案件脉络。

## 排序原则

### 一、时间优先

**基本原则**：按时间从早到晚排序

**排序规则**：
- 明确时间：按具体日期排序
- 模糊时间：根据时间区间确定大致位置
- 推断时间：按推断时间排序，在内容中注明
- 缺失时间：统一置于表格末尾

### 二、同日处理

多个事件发生在同一天时：

**排序方式**：
1. 按事件发生先后顺序（如有明确时间点）
2. 按事件重要性（核心事件在前）
3. 按事件类型（法律行为在前，沟通行为在后）

**示例**：
- 同日既有合同签订又有款项支付
- 合同签订在前，款项支付在后

### 三、模糊时间处理

**时间段表述的处理**：
| 表述方式 | 排序位置 |
|----------|----------|
| 年初 | 该年1月 |
| 年中 | 该年6-7月 |
| 年末 | 该年12月 |
| 月初 | 该月1-10日 |
| 月中 | 该月11-20日 |
| 月末 | 该月21-31日 |

**相对时间处理**：
- "合同签订后一周"：根据基准事件时间计算
- "判决生效后15日内"：根据判决生效时间计算

### 四、时间未知处理

**统一规则**：
- 时间列标注"时间未知"
- 排在表格最末尾
- 多个时间未知的事件按事件类型排序

## 排序步骤

1. **提取时间**：从所有事件中提取时间信息
2. **标准化处理**：统一时间格式
3. **确定位置**：根据时间确定排序位置
4. **处理特殊情况**：同日、模糊时间、缺失时间
5. **生成表格**：按排序输出

## 时间序列验证

排序完成后需验证时间序列合理性：

### 逻辑验证

检查事件之间的逻辑关系：
- 合同签订应早于合同履行
- 起诉应早于开庭
- 一审判决应早于二审

### 因果验证

检查有因果关系的事件顺序：
- 催告应在要求履行之前
- 违约通知应在主张违约责任之前

### 发现矛盾时的处理

1. 核实材料记载是否准确
2. 检查是否存在时间推断错误
3. 如实保留矛盾情况，在内容中说明

## 输出格式

### 标准格式

```markdown
# 案件大事记

| 时间 | 事件 | 主要内容 |
|------|------|----------|
| 2023年1月15日 | 合同签订 | ... |
| 2023年2月1日 | 履约行为 | ... |
| 2023年3月15日 | 程序事件 | ... |
| 时间未知 | 沟通事件 | ... |
```

### 分段格式（事件较多时）

可按时间阶段分段，在表格前添加阶段标题：

```markdown
## 一、合同签订阶段

| 时间 | 事件 | 主要内容 |
|------|------|----------|
| ... | ... | ... |

## 二、合同履行阶段

| 时间 | 事件 | 主要内容 |
|------|------|----------|
| ... | ... | ... |
```
