# 事件分类法

对案件中的事件进行分类，有助于理解事件性质和后续分析。

## 事件类型

### 一、法律行为

**定义**：基于当事人意思表示，产生法律效果的行为

**常见类型**：

| 类型 | 示例 |
|------|------|
| 合同行为 | 签订合同、变更合同、解除合同 |
| 履约行为 | 交付货物、支付价款、提供服务 |
| 违约行为 | 逾期付款、质量瑕疵、拒绝履行 |
| 处分行为 | 转让财产、设立抵押、变更登记 |
| 授权行为 | 委托代理、授权签字 |

### 二、法律事件

**定义**：不以当事人意志为转移，但产生法律效果的事实

**常见类型**：

| 类型 | 示例 |
|------|------|
| 自然事件 | 地震、洪水、台风等不可抗力 |
| 意外事件 | 交通事故、意外伤害 |
| 时间经过 | 诉讼时效届满、除斥期间届满 |
| 身份事件 | 出生、死亡、失踪 |

### 三、程序事件

**定义**：诉讼或仲裁程序中的事件

**常见类型**：

| 类型 | 示例 |
|------|------|
| 起诉类 | 提起诉讼、反诉、撤诉 |
| 开庭类 | 立案、开庭、休庭 |
| 裁判类 | 判决、裁定、调解 |
| 上诉类 | 上诉、二审、再审 |
| 执行类 | 申请执行、执行终结 |

### 四、沟通事件

**定义**：当事人之间的意思传达行为

**常见类型**：

| 类型 | 示例 |
|------|------|
| 通知 | 发送解除通知、送达通知函 |
| 催告 | 催告履行、催款函 |
| 协商 | 谈判、调解沟通 |
| 承诺 | 同意履行、认可债务 |

## 分类原则

### 主次原则

一个事件可能涉及多种性质时，以主要性质归类：
- 示例："起诉并申请财产保全"归类为"起诉类程序事件"

### 效果优先原则

优先考虑事件的法律效果而非行为形式：
- 示例："发函催告付款"虽为沟通行为，但产生诉讼时效中断效果，可归入法律事件

### 关联性原则

与案件争议焦点相关的事件优先归类：
- 示例：合同纠纷中，与合同签订履行相关的事件为重点

## 分类步骤

1. **识别事件**：从材料中提取具体事件
2. **判断性质**：分析事件的法律性质
3. **确定类型**：归入相应类型
4. **标注事件**：在"事件"列填写类型简称

## 事件列填写规范

使用简洁的类型标识：

| 类型标识 | 含义 |
|----------|------|
| 合同签订 | 签订合同 |
| 履约行为 | 按约履行 |
| 违约行为 | 违反约定 |
| 程序事件 | 诉讼程序事项 |
| 沟通事件 | 意思传达 |
| 不可抗力 | 自然灾害等 |
| 其他 | 其他事件 |

**示例**：
- 合同签订、首付款支付、过户登记、催告、纠纷产生、起诉、开庭、判决
