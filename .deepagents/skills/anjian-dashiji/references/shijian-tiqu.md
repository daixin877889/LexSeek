# 时间提取法

从案情材料中准确识别和提取时间信息，确保时间线的完整性。

## 时间类型

### 一、明确时间

**定义**：材料中直接记载的具体日期

**识别信号**：
- 具体日期：2023年1月15日、2023.01.15、2023/1/15
- 节日表述：春节、国庆节期间（需转换为具体日期）
- 法律期限：判决生效后15日内（需计算具体日期）

**处理方式**：统一格式为"YYYY年MM月DD日"

**示例**：
- 原文："2023.1.15双方签订合同"
- 提取：2023年1月15日

### 二、模糊时间

**定义**：材料中使用时间区间或相对时间表述

**类型及处理**：

| 类型 | 示例 | 处理方式 |
|------|------|----------|
| 时间段 | 2023年年初 | 如实表述，保留模糊性 |
| 相对时间 | 合同签订后一周 | 根据基准时间计算 |
| 季节表述 | 当年春季 | 如实表述 |
| 时间范围 | 3月至5月间 | 如实表述 |

### 三、推断时间

**定义**：根据事件序列逻辑推断的时间

**推断原则**：
- 必须有明确的推断依据
- 在主要内容中注明推断依据
- 不确定的推断使用"约""前后"等限定词

**示例**：
- 原文："合同签订后，甲依约交付货物，乙收货后一周内付款"
- 推断：已知合同签订日期为1月15日，推断交付时间约1月中下旬，付款时间约1月底

### 四、缺失时间

**定义**：材料中未提及时间的事件

**处理规则**：
- 时间列标注"时间未知"
- 该类事件排在表格末尾
- 不要凭空猜测或虚构时间

**示例**：
- 原文："双方曾就合同条款进行协商"
- 处理：时间列填写"时间未知"

## 提取步骤

1. **通读材料**：整体把握案情脉络
2. **标记时间**：识别所有时间表述
3. **判断类型**：明确/模糊/推断/缺失
4. **标准化处理**：统一格式，计算相对时间
5. **交叉验证**：检查时间序列是否合理

## 常见问题处理

### 时间冲突

不同材料记载的时间不一致时：
1. 优先采纳证据效力更高的材料
2. 如实记录冲突情况
3. 在主要内容中注明存在争议

### 时间顺序矛盾

后发生的事件记载的时间早于前发生事件：
1. 检查是否为材料记载错误
2. 核实事件实际发生顺序
3. 按实际发生顺序排列，在主要内容中说明

### 时间表述不规范

材料中时间表述模糊或不完整：
1. 尽可能根据上下文推断
2. 无法推断的如实保留模糊表述
3. 年份缺失时根据案情推断

## 时间节点类型清单

为确保时间节点不遗漏，按以下类型主动搜寻：

### 合同类纠纷时间节点

| 节点类型 | 检查要点 |
|----------|----------|
| 合同成立 | 签订时间、签订地点 |
| 合同生效 | 生效条件成就时间、生效日期约定 |
| 履行期限 | 约定的履行时间节点 |
| 实际履行 | 交付、付款、验收等时间 |
| 违约行为 | 违约发生时间、持续状态 |
| 催告通知 | 催告发出/到达时间 |
| 解除撤销 | 解除通知发出/到达时间 |
| 争议产生 | 纠纷发生时间 |

### 侵权类纠纷时间节点

| 节点类型 | 检查要点 |
|----------|----------|
| 侵权行为 | 行为发生时间、持续期间 |
| 损害结果 | 损害发生时间、损害扩大时间 |
| 因果链 | 各环节发生时间 |
| 权利知晓 | 知道或应当知道权利受损时间 |
| 维权行为 | 投诉、举报、协商时间 |

### 物权类纠纷时间节点

| 节点类型 | 检查要点 |
|----------|----------|
| 权利取得 | 取得时间、登记时间 |
| 权利变动 | 转让、变更时间 |
| 占有状态 | 占有开始/结束时间 |
| 妨害行为 | 妨害发生时间 |

### 程序类事件时间节点

| 节点类型 | 检查要点 |
|----------|----------|
| 起诉立案 | 起诉时间、立案时间 |
| 开庭审理 | 开庭时间、次数 |
| 诉讼行为 | 举证、质证、辩论时间 |
| 裁判文书 | 判决/裁定时间、送达时间 |
| 上诉再审 | 上诉时间、再审申请时间 |
| 执行程序 | 申请执行时间、执行结案时间 |

### 时效期间节点

| 节点类型 | 检查要点 |
|----------|----------|
| 诉讼时效 | 起算点、中断事由时间 |
| 除斥期间 | 起算点、届满时间 |
| 履行期限 | 约定期限、合理期限 |

## 完整性检查

提取时间节点后，按以下步骤检查完整性：

1. **类型遍历**：对照上述节点类型清单，逐一检查是否有遗漏
2. **因果验证**：检查有因果关系的事件是否都有时间记录
3. **阶段衔接**：检查各阶段之间是否有时间断层
4. **矛盾排查**：检查时间冲突或逻辑矛盾
