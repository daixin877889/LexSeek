# 案件概要整理方法论

## 简介

本skill提供法律案件概要的系统整理方法，用于：

- 将散乱的案情材料整理为结构化概要
- 从判决书中提取关键信息和法律条款
- 保持信息客观性和完整性
- 为后续法律分析提供基础

## 使用场景

**适用场景**：
- 律师整理案件材料
- 法官梳理案情要点
- 法律工作者撰写案情摘要
- 判决书信息提取

**不适用场景**：
- 刑事案件（结构不同）
- 行政案件（结构不同）
- 需要法律分析的场景
- 需要预测判决结果的场景

## 快速使用指南

### 最简流程（普通材料）

1. 阅读材料，识别当事人和案件性质
2. 按概要结构整理信息
3. 标注信息缺失项
4. 保持原文概念锁定

### 完整流程（判决书）

1. 确认材料为判决书
2. 提取案件基本信息和当事人
3. 整理案件事实和争议焦点
4. 列出引用的法律条款名称
5. 按结构输出完整概要
注：法条全文检索由请求权基础模块负责

## 与其他模块配合

本skill为事实整理层，服务于后续分析模块：

- **前置**：用户输入案情材料
- **本模块**：整理结构化概要（简要叙述）
- **大事记模块**：整理详细时间线（表格形式）
- **后续模块**：法律关系分析、请求权基础分析、案由选择

**与大事记模块的分工**：

| 模块 | 内容 | 格式 |
|------|------|------|
| **案件概要** | 简要叙述案件事实 | 一段话（200-400字） |
| **大事记** | 详细时间线梳理 | 表格（时间、事件、主要内容） |

案件概要提供"全景图"，大事记提供"时间轴"，两者配合使用。

输出特征：
- 案情简要摘要
- 争议焦点四方面分类
- （判决书场景）法律条款清单
