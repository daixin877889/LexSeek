# 案件概要整理方法论

## 核心原则

案件概要整理以"客观呈现、完整提取、概念锁定"为核心原则，将用户提供的材料整合为一份符合法律行业专业表述的结构化概要。

整理方法遵循三个基本原则：

1. **客观性原则**：只整理材料中明确存在的信息，不杜撰、不推断、不补充
2. **完整性原则**：按照标准结构全面提取信息，无信息处明确标注"暂无相关信息"
3. **概念锁定原则**：保持原文概念原貌，禁止擅自具体化、实例化抽象概念

## 关键要点

### 一、概要结构
详见 [references/概要结构.md](./references/概要结构.md)

标准概要包含以下部分（按需呈现）：
- 案件基本信息
- 当事人信息
- 案件事实
- 争议焦点
- 诉讼请求/答辩要点
- 法律条款（仅判决书）

### 二、事实整理
详见 [references/事实整理.md](./references/事实整理.md)

采用简要叙述格式，而非详细时间线（详细时间线由大事记模块处理）：
- 以一段话简要叙述案件事实
- 包含当事人关系、交易性质、核心义务、关键行为、争议起因
- 篇幅控制在200-400字

### 三、判决书处理
详见 [references/判决书处理.md](./references/判决书处理.md)

识别判决书的方法：
- 文书名称判断
- 文书结构识别
- 关键要素确认
- 审级识别（一审/二审/再审）

判决书特殊处理：
- 按审级规则呈现诉讼请求/答辩：一审直接列原告诉讼请求和被告答辩；二审先简述一审判决再列上诉请求和答辩；再审类推
- 提取引用的法律条款名称清单
- 不调用法条检索工具（全文检索由请求权基础模块负责）

### 四、争议焦点提取
详见 [references/争议焦点.md](./references/争议焦点.md)

争议焦点的四个方面：
- 证据方面：证据资格、非法证据、举证责任
- 要件事实：事实认定、合同效力、要件成就
- 法律适用：法律冲突、新旧法衔接、法律解释
- 程序方面：程序权利、程序错误、审理范围

争议焦点提取流程：
- 梳理双方诉求（诉讼请求vs答辩意见）
- 区分承认与不承认
- 区分否认vs抗辩（举证责任不同）
- 梳理一致点和分歧点
- 对分歧点分类归入四个方面
- 检验是否超出审理范围

### 五、概念锁定
详见 [references/概念锁定.md](./references/概念锁定.md)

处理抽象概念的核心规则：
- 保持概念原貌
- 禁止概念滑坡
- 禁止擅自实例化

## 使用流程

```
材料输入
    │
    ├──→ 判断材料类型
    │       │
    │       ├── 是判决书 → 按判决书流程处理
    │       │               │
    │       │               ├── 整理案情事实
    │       │               └── 提取法律条款名称清单
    │       │
    │       └── 非判决书 → 按普通材料流程处理
    │                       │
    │                       └── 整理客观事实
    │
    └──→ 按概要结构输出
```

## 适用范围

**适用场景**：
- 民事案件材料整理
- 商事案件材料整理
- 劳动争议案件材料整理
- 判决书信息提取

**不适用场景**：
- 刑事案件（结构不同）
- 行政案件（结构不同）
- 需要法律分析的案件
- 需要预测判决结果的案件

## 与其他模块配合

本skill为事实整理层，服务于后续分析模块：

- **前置**：用户输入案情材料
- **本模块**：整理结构化概要
- **后续模块**：法律关系分析、请求权基础分析、案由选择

输出特征：
- 案情结构化摘要
- 关键事实标签
- 时间线节点
- （判决书场景）法律条款清单
