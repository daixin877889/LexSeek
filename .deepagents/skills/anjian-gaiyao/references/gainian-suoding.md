# 概念锁定法

处理案件材料中抽象概念的规范方法，确保信息整理的客观性。

## 核心原则

**原文概念锁定原则**：对于案件材料中的任何抽象或集合性概念，必须保持其原貌，不得擅自具体化、实例化或进行联想。

## 概念类型识别

### 抽象概念

需要锁定的抽象概念类型：

| 概念类型 | 示例 | 错误处理（禁止） | 正确处理 |
|----------|------|------------------|----------|
| 损失类 | "相关损失" | 具体化为"财产损失" | 保持"相关损失" |
| 行为类 | "不当行为" | 具体化为"侵权行为" | 保持"不当行为" |
| 费用类 | "合理费用" | 具体化为"交通费、律师费" | 保持"合理费用" |
| 损害类 | "各种损害" | 列举具体损害类型 | 保持"各种损害" |
| 金额类 | "一定金额" | 推断具体数额 | 保持"一定金额" |

### 集合概念

需要锁定的集合概念类型：

| 概念类型 | 示例 | 错误处理（禁止） | 正确处理 |
|----------|------|------------------|----------|
| 物品类 | "相关物品" | 列举具体物品 | 保持"相关物品" |
| 人员类 | "相关人员" | 列举具体人员 | 保持"相关人员" |
| 文件类 | "相关文件" | 列举具体文件 | 保持"相关文件" |
| 证据类 | "相关证据" | 列举具体证据 | 保持"相关证据" |

## 锁定规则

### 规则一：禁止概念滑坡

**定义**：不得将抽象概念在整理过程中降级为具体概念。

**错误示例**：
```markdown
原文：原告主张被告的不当行为造成了相关损失。
错误整理：原告主张被告的侵权行为造成了财产损失和精神损害。
```

**正确示例**：
```markdown
原文：原告主张被告的不当行为造成了相关损失。
正确整理：原告主张被告的不当行为造成了相关损失。
```

### 规则二：保持概念原貌

**定义**：使用材料原文中的表述，不增加、不删减、不解释概念的内涵。

**错误示例**：
```markdown
原文：双方就相关费用产生争议。
错误整理：双方就交通费、住宿费、律师费等费用产生争议。
```

**正确示例**：
```markdown
原文：双方就相关费用产生争议。
正确整理：双方就相关费用产生争议。
```

### 规则三：禁止推断补充

**定义**：不得根据上下文推断概念的具体内容并补充到概要中。

**错误示例**：
```markdown
原文：原告提交了相关证据证明其主张。
错误整理：原告提交了合同、转账记录、聊天记录等证据证明其主张。
（注：即使可以推断证据类型，也不得补充）
```

**正确示例**：
```markdown
原文：原告提交了相关证据证明其主张。
正确整理：原告提交了相关证据证明其主张。
```

## 特殊情况处理

### 当事人具体化了概念

如果材料中当事人或法院已经具体化了概念，可以如实记录：

```markdown
原文：原告主张相关损失共计人民币5万元，包括医疗费3万元和误工费2万元。
正确整理：原告主张相关损失共计人民币5万元，包括医疗费3万元和误工费2万元。
（注：原文已经具体化，如实整理）
```

### 概念有上下文解释

如果材料中其他部分对概念有解释，可以引用解释：

```markdown
原文第1段：原告主张被告赔偿相关损失。
原文第2段：相关损失包括：医疗费3万元、误工费2万元。

正确整理：原告主张被告赔偿相关损失。相关损失包括：医疗费3万元、误工费2万元。
（注：引用原文解释，而非自行推断）
```

### 概念存在争议

如果当事人对概念内容有争议，分别记录各方主张：

```markdown
原告主张相关损失包括医疗费、误工费、护理费。
被告认为相关损失仅限于医疗费。
法院认为...
```

## 检查清单

整理完成后，检查以下项目：

- [ ] 所有"相关XX"是否保持原貌？
- [ ] 所有"不当XX"是否保持原貌？
- [ ] 所有"合理XX"是否保持原貌？
- [ ] 所有"一定XX"是否保持原貌？
- [ ] 是否有自行推断的具体内容？
- [ ] 是否有未经原文支持的概念展开？

如有违反，必须修改。
