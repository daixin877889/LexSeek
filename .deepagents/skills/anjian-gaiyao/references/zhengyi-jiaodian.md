# 争议焦点提取法

从案件材料中系统提取争议焦点的规范方法。

## 核心原则

**简明固定原则**：争议焦点归纳要求简明、固定，是本案庭审的审理对象。

## 争议焦点的四个方面

### 一、证据方面

证据类争议焦点**仅涉及证据本身的资格和效力**，不涉及对法律关系或合同条款的实质判断。

| 属于证据方面 | 不属于证据方面（属要件事实） |
|-------------|---------------------------|
| 某项证据能否采信 | 某份合同的效力如何 |
| 证据是否应作为非法证据排除 | 某协议的约定应如何解释 |
| 举证责任是否分配错误 | 某一行为是否构成违约 |
| 证据是否达到证明标准 | 当事人之间的法律关系性质 |

**提取信号**：
- "该证据不应采信"
- "举证责任应由对方承担"
- "该证据系非法取得"

**单方陈述/咨询场景的处理**：
当材料仅为当事人单方陈述（非判决书、非证据清单）时，证据方面不写具体证据争议，而是：
- 指出当前信息源于单方陈述，关键事实需证据支撑
- 列出哪些核心事实在后续程序中需要重点举证
- 不在此场景下写"暂无相关信息"，因为陈述本身就是信息源

**格式（单方陈述场景）**：
```markdown
### 一、证据方面

> 当前信息来源为当事人单方陈述。后续程序中，以下事实需重点举证：
1. [核心事实1的证明方向]
2. [核心事实2的证明方向]
```

### 二、要件事实方面

要件事实类争议焦点涉及：

| 类型 | 具体内容 |
|------|----------|
| 事实认定 | 谁违约、是否构成某行为 |
| 合同效力 | 合同是否有效、是否可撤销 |
| 要件成就 | 是否构成要件（如"是否构成较重婚外情"） |

**提取信号**：
- "原告主张...被告抗辩..."
- "是否构成..."
- "合同是否有效"

**合同效力特别注意**：
- 涉及金融安全的合同效力认定需详细论述
- 违反规章是否触发金融安全需具体论证
- 参考《九民会纪要》相关判断标准

### 三、法律适用方面

法律适用类争议焦点涉及：

| 类型 | 具体内容 |
|------|----------|
| 法律冲突 | 不同法律规定之间的冲突 |
| 新旧法衔接 | 新法与旧法的适用问题 |
| 法律解释 | 对法律条文的不同理解 |

**提取信号**：
- "应适用...法而非...法"
- "...条与...条冲突"
- "对该条款应理解为..."

**常见情形**：
- 《公司法》与《民法典》的衔接（如清算义务人问题）
- 《物权法》与《婚姻法》的衔接（如夫妻间物权变动）
- 新司法解释与旧法的适用

**法规名称引用规则**：
- **判决书场景**：争议焦点中提及的法规名称，须是判决书原文中已出现的，不得自行引入
- **非判决书场景**：争议焦点的法律适用问题只描述"法律冲突是什么"，不引用具体法规名称。例如写"新旧法适用标准之争"而非"应适用《XX法》还是《XX法》"
- 具体法规的检索和确定由请求权基础模块完成

### 四、程序方面

程序类争议焦点涉及：

| 类型 | 具体内容 |
|------|----------|
| 程序权利 | 能否上诉、是否有答辩期 |
| 程序错误 | 程序是否违法 |
| 审理范围 | 是否超出审理范围 |

**提取信号**：
- "程序违法"
- "应给我答辩期"
- "超出审理范围"

**输出规则**：
- 程序方面的问题不能只写标签（如"工伤认定后的救济路径"），必须写明具体的程序路径节点
- 例如：写"工伤认定被拒 → 申请劳动仲裁确认劳动关系 → 凭仲裁裁决申请工伤认定"，而非"工伤认定的救济路径问题"
- 如材料中仅有程序问题的方向但未明确具体路径，写清已明确的节点即可

## 争议焦点提取流程

### 第一步：梳理双方诉求

```markdown
原告诉讼请求：
1. ...
2. ...

被告答辩要点：
1. ...
2. ...
```

### 第二步：区分承认与不承认

**承认**：双方一致的内容 → 不是争议焦点

**不承认**：进一步区分：

| 类型 | 定义 | 举证责任 |
|------|------|----------|
| **否认** | 否定对方主张的事实 | 不负举证责任 |
| **抗辩** | 提出新事实抵消对方主张 | 负举证责任 |

**判断方法**：
- 简单说"不是""没有""不存在" → 否认
- 提出"因为...所以不成立" → 抗辩

### 第三步：梳理一致点和分歧点

```markdown
一致点（无争议）：
- 双方认可合同成立
- 双方认可已支付5万元
- ...

分歧点（有争议）：
- 原告主张已付全款，被告主张仅付定金
- 原告主张被告违约，被告主张原告违约
- ...
```

### 第四步：对分歧点分类

将每个分歧点归入争议焦点的四个方面：

```markdown
分歧点 → 争议焦点分类

"该转账记录能否证明付款" → 证据方面
"谁违约" → 要件事实方面
"合同是否有效" → 要件事实方面（合同效力）
"应适用民法典还是合同法" → 法律适用方面
```

### 第五步：检验是否超出审理范围

**核心检查**：争议焦点是否在案由的要件事实范围内

**排除规则**：
- 不属于该案由要件事实的 → 不纳入争议焦点
- 超出审理范围的 → 不纳入争议焦点

**示例**：
藏獒案（烈性犬）中，"受害人是否挑逗藏獒"不是要件事实，不纳入争议焦点

## 输出格式

### 标准输出结构

```markdown
## 争议焦点

### 一、证据方面
1. [证据类争议焦点1]
2. [证据类争议焦点2]

### 二、要件事实方面
1. [事实类争议焦点1]
2. [事实类争议焦点2]

### 三、法律适用方面
1. [法律适用类争议焦点]
（如有）

### 四、程序方面
1. [程序类争议焦点]
（如有）
```

### 简化输出结构（无分类时）

```markdown
## 争议焦点

1. [争议焦点1]
2. [争议焦点2]
3. [争议焦点3]
```

## 注意事项

### 一、不得超出审理范围

争议焦点必须在案由的要件事实范围内：
- 参照案由书确定审理范围
- 不属于该案由要件的事实不纳入

### 二、区分否认与抗辩

这是判断举证责任的关键：
- 否认：直接否定，不负举证责任
- 抗辩：提出新事实，负举证责任

### 三、合同成立与生效的举证责任

根据新证据规定：
- 主张合同成立 → 原告举证
- 主张合同无效 → 被告举证（或法官依职权审查）

### 四、保持简明固定

每个争议焦点应是：
- 明确的审理对象
- 简短的表述
- 固定的范围

## 检查清单

提取完成后检查：

- [ ] 是否覆盖了双方的主要分歧？
- [ ] 是否区分了承认和不承认？
- [ ] 是否区分了否认和抗辩？
- [ ] 是否归入了四个方面？
- [ ] 是否超出审理范围？
- [ ] 每个焦点是否简明固定？
- [ ] 是否遗漏明显争议？
