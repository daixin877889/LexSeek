# 判决书处理法

识别判决书并提取法律条款的系统方法。

## 判决书识别

### 识别要素

判断材料是否为判决书，检查以下要素：

**文书名称判断**：
- 名称包含"判决书"、"裁定书"、"调解书"
- 典型格式：XX人民法院民事判决书

**文书结构识别**：
- 开头：法院名称、文书类型、案号
- 中部：当事人信息、案件事实、争议焦点
- 尾部：判决结果、法律依据、审判人员署名

**关键要素确认**：
- 案号（如：(2023)X民初X号）
- 当事人（原告、被告、第三人）
- 判决结果（判决如下...）
- 审判人员（审判长、审判员、书记员）

### 判断流程

```
材料输入
    │
    ├── 文书名称检查
    │   └── 包含"判决书"等关键词？
    │
    ├── 文书结构检查
    │   └── 包含案号、当事人、判决结果？
    │
    └── 关键要素检查
        └── 包含审判人员署名？
    │
    └── 三项均符合 → 确认为判决书
        任一项不符 → 非判决书，按普通材料处理
```

### 审级识别

确认为判决书后，进一步识别审级：

**识别要素**：
- 文书名称中含"初"字（如民初X号） → 一审判决书
- 文书名称中含"终"字（如民终X号） → 二审判决书
- 文书名称中含"再"字（如民再X号） → 再审判决书
- 法院层级：基层院、中级院多为一审，高级院、最高院可能是一审或上诉审

**审级决定诉讼请求/答辩要点的呈现方式**：

| 审级 | 呈现方式 |
|------|----------|
| **一审判决书** | 直接列"原告诉讼请求"和"被告答辩要点" |
| **二审判决书** | 先简述一审判决结果（一段话），再列"上诉人上诉请求"和"被上诉人答辩要点" |
| **再审判决书** | 先简述原审判决结果，再列"再审申请人请求"和"被申请人答辩要点" |

## 判决书特殊处理

### 一、案情整理

判决书的案情整理与其他材料相同，按照概要结构法执行。

### 二、法律条款提取

**提取范围**：
- 判决书"本院认为"部分引用的法条
- 判决结果部分引用的法条
- 当事人主张中引用的法条（如判决书有记载）

**提取格式**：

```markdown
## 判决书引用的法律条款

### 判决书原文引用
1. 《中华人民共和国民法典》第一百四十三条
2. 《中华人民共和国民法典》第五百零九条
3. 《中华人民共和国民事诉讼法》第六十七条
...
```

### 三、法条全文检索规则

**本模块不负责法条全文检索。** 法条全文检索是请求权基础模块的职责。

- 判决书场景仅列出引用的法条名称，不调用法条检索工具
- 不展示法条全文
- 法条具体内容留待请求权基础模块在规范检索阶段获取

## 非判决书的处理

当材料确认为非判决书时：

**禁止事项**：
- 不调用法条检索工具
- 不自行检索相关法律条文
- 不联想可能适用的法律

**正确行为**：
- 仅整理材料中的客观事实
- 法律条文的查找与分析工作由后续模块处理

## 处理流程图

```
材料输入
    │
    ├──→ 判断材料类型
    │       │
    │       ├── 是判决书
    │       │       │
    │       │       ├── 识别审级（一审/二审/再审）
    │       │       ├── 整理案情事实
    │       │       ├── 按审级规则提取诉讼请求/答辩
    │       │       └── 提取法律条款名称清单
    │       │
    │       └── 非判决书
    │               │
    │               └── 整理客观事实
    │
    └──→ 输出结构化概要
```
