# 概要结构法

案件概要的标准结构组成部分。根据材料内容按需呈现，不强制包含所有部分。

## 标准结构

### 一、案件基本信息

```markdown
## 案件基本信息

- 案由：[根据材料填写或暂无]
- 案号：[如材料中有]
- 受理法院：[如材料中有]
- 案件阶段：[一审/二审/执行等]
- 案件类型：[民事/商事/劳动]
```

### 二、当事人信息

**格式要求**：
- 使用列表格式呈现，**禁止使用表格**
- 按诉讼立场分组（原告方/被告方），非按当事人逐一展开
- 同一立场下有多个当事人时，在该分组下紧凑列出，不重复分组标题
- 每组只用一个 `###` 标题

**非判决书场景模板**：

```markdown
## 当事人信息

### 原告方
- 当事人：[姓名/名称]，[性别]，[年龄]
- 身份：[职业/角色]
- 主张：[核心诉求概述]

### 被告方
- 当事人：[姓名/名称]
- 身份：[职业/角色]
- 主张：[核心抗辩概述]

### 第三人（如有）
- 当事人：[姓名/名称]，[身份]
```

**判决书场景**：按判决书自身的当事人分类呈现（上诉方/被上诉方，或再审申请人/被申请人等），分组原则同上。

**分组原则**：
- 简单案件（≤3方）：原告方/被告方/第三人，每方最多一个 `###` 区块
- 复杂案件（多当事人）：同一诉讼立场下的多个当事人放在同一个 `###` 区块下，用段落间距区分，不为每个当事人单独设 `###` 标题
- 判决书场景：按判决书中的当事人程序身份命名分组标题（如"上诉人"、"被上诉人"、"一审被告"），不按法律关系角色自行命名

**非当事人关联方的处理**：
- 仅在案件事实中涉及、但非本案当事人或未列为被申请人的主体（如合同相对方、中介方、第三方平台等），不列入"当事人信息"区块
- 该等主体在案件事实叙述中说明其角色和与当事人的关系即可
- 判断标准：该主体是否在本案中被列为当事人（诉讼/仲裁中）或申请人明确提出针对其的请求？是→列入当事人信息；否→仅在事实段落出现

```markdown
## 案件事实

[以一段话简要叙述案件事实，包含当事人关系、交易性质、核心义务、关键行为、争议起因]

示例：
碧桂园公司与凯利公司于2017年7月15日签订《资产转让合同》，约定凯利公司将三亚蓝月湾海景酒店公寓项目转让给碧桂园公司，转让价款暂定7亿元。碧桂园公司支付诚意金3.2亿元后，凯利公司未能在约定期限完成规划调整，主张因政府政策调整属于不可抗力。碧桂园公司发函解除合同并要求退还诚意金及违约金。
```

**注意**：案件事实采用简要叙述格式，详细时间线由大事记模块处理。

### 四、争议焦点

详见 [争议焦点.md](./争议焦点.md) 的完整方法。

```markdown
## 争议焦点

### 一、证据方面
1. [证据类争议焦点]
（如有）

### 二、要件事实方面
1. [事实类争议焦点]
（如有）

### 三、法律适用方面
1. [法律适用类争议焦点]
（如有）

### 四、程序方面
1. [程序类争议焦点]
（如有）
```

简化格式（无明显分类时）：

```markdown
## 争议焦点

1. [争议焦点1]
2. [争议焦点2]
...
```

### 五、诉讼请求（原告方）

```markdown
## 诉讼请求

1. [请求1]
2. [请求2]
...

### 事实与理由摘要
[原告主张的事实和理由概述]
```

### 六、答辩要点（被告方，如有）

```markdown
## 答辩要点

1. [答辩要点1]
2. [答辩要点2]
...

### 答辩理由摘要
[被告主张的事实和理由概述]
```

### 七、法律条款（仅判决书场景）

```markdown
## 判决书引用的法律条款清单

1. 《[法律名称]》第X条（一审引用）
2. 《[法律名称]》第X条（二审引用）
...
```
注：仅列法条名称，不查全文。法条全文检索由请求权基础模块负责。

### 八、争议核心

位于概要末尾，一段话点明本案的核心法律问题和实务影响。不是法律分析或预测，是对已有信息的价值压缩。

**写法**：
- 指出本案争议的关键点（如"本案核心在于XX是否构成YY"）
- 点明不同结论下各通向哪里（如"若构成→A路径；若不构成→B路径"）
- 控制在30-50字

```markdown
## 争议核心

本案核心在于[争议事项]是否构成[法律定性]。若构成，则[后果A]；若不构成，则仅能[后果B]。
```

## 结构使用原则

1. **按需呈现**：材料有对应信息才呈现该部分
2. **信息缺失标注**：某项无信息时明确标注"暂无相关信息"
3. **保持顺序**：按上述标准顺序组织各部分
4. **层级清晰**：使用Markdown标题层级区分

## 信息缺失处理

当某部分或某项信息在材料中不存在时：

```markdown
## [部分名称]

根据现有材料，暂无相关信息。
```

或单项缺失：

```markdown
- 案号：根据现有材料，暂无相关信息
```
