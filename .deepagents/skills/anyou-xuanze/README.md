# 案由选择方法论

## 简介

本skill提供民事案件案由的系统选择方法，用于：

- 根据案情特征标签检索匹配案由
- 校验案由版本是否为最新
- 处理案由竞合与冲突
- 确定最终适用案由

## 使用场景

**适用场景**：
- 律师确定案件案由
- 法官审查案由是否准确
- 法律工作者撰写案由分析报告
- 案由变更合理性审查

**不适用场景**：
- 刑事案件（案由体系不同）
- 行政案件（案由体系不同）
- 程序性问题分析

## 快速使用指南

### 最简流程

1. 获取案情特征标签和主要规范
2. 构造检索关键词检索《民事案件案由规定》
3. 校验版本，选择最新版案由
4. 按优先级规则确定最终案由

### 完整流程

1. 获取请求权分析模块输出
2. 第一轮精确检索案情特征标签对应案由
3. 若无结果，第二轮扩大检索相邻领域
4. 若仍无结果，第三轮检索通用案由
5. 校验候选案由的生效日期
6. 按优先级规则排序候选案由
7. 处理可能的冲突情形
8. 验证最终案由存在于《民事案件案由规定》
9. 输出案由分析报告

## 与其他工具配合

本skill为方法论框架，实际选择时应配合：

- **法条检索工具**（如search_law）：检索《民事案件案由规定》
- **请求权基础分析模块**：提供案情特征标签和主要规范

使用流程：
1. 从请求权分析模块获取输入
2. 使用法条检索工具检索案由
3. 使用本skill进行案由分析与选择
4. 使用法条检索工具验证最终案由
