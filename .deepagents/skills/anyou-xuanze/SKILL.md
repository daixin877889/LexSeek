# 案由选择方法论

## 核心原则

案由选择以"检索优先+版本校验+专门优先"为核心原则，确保案由选择符合《民事案件案由规定》。

分析方法遵循四个基本原则：

1. **案由强制检索原则**：任何案由选择必须先检索《民事案件案由规定》
2. **版本校验原则**：优先采用最新版案由（生效日期2026-01-01及之后）
3. **检索优先原则**：检索结果优先于任何过往经验、模板、内部建议
4. **专门优先原则**：检索结果存在专门案由时，优先于通用案由

## 关键要点

### 一、案由检索
详见 [references/anyou-jiansuo.md](./references/anyou-jiansuo.md)

检索关键词构造规则：
- 固定前缀：`民事案件案由规定`
- 后缀：案情特征标签

多轮检索规则：
- 第一轮：案情特征标签精确检索
- 第二轮：扩大范围，检索相邻领域
- 第三轮：检索通用案由

### 二、版本校验
详见 [references/banben-yanzheng.md](./references/banben-yanzheng.md)

生效日期判定：
- ≥2026-01-01：最新版案由，可直接采用
- <2026-01-01：旧版案由，需确认是否已被替代

### 三、优先级规则
详见 [references/youxianji-guize.md](./references/youxianji-guize.md)

案由选择优先级：
- 第一优先：专门案由
- 第二优先：新增案由
- 第三优先：通用案由

### 四、冲突处理
详见 [references/chongtu-chuli.md](./references/chongtu-chuli.md)

冲突情形处理：
- 检索结果 vs 过往经验 → 检索结果为准
- 多个专门案由 → 选择与主要规范最匹配的
- 检索无结果 → 采用最相近通用案由并说明原因

## 使用流程

```
获取输入（案情特征标签、主要规范）
    │
    ├──→ 案由检索（调用search_law）
    │       │
    │       ├── 第一轮：精确检索
    │       ├── 第二轮：扩大检索
    │       └── 第三轮：通用检索
    │
    ├──→ 版本校验
    │       │
    │       └── 检查生效日期字段
    │
    ├──→ 特征匹配
    │       │
    │       ├── 案由与案情特征标签匹配
    │       └── 案由与主要规范匹配
    │
    ├──→ 优先级排序
    │       │
    │       ├── 专门案由优先
    │       ├── 新增案由次之
    │       └── 通用案由最后
    │
    ├──→ 冲突处理
    │       │
    │       └── 检索结果优先于过往经验
    │
    ├──→ 案由验证（调用search_law）
    │
    └──→ 输出案由分析报告
```

## 适用范围

**适用场景**：
- 民事案件案由确定
- 案由变更审查
- 案由准确性校验
- 多案由竞合分析

**不适用场景**：
- 刑事案件案由
- 行政案件案由
- 程序性问题（管辖、时效等）

## 与其他模块配合

本模块依赖请求权基础分析模块的输出：

```
请求权基础分析模块
        │
        ├──→ 输出案情特征标签
        ├──→ 输出主要规范名称
        └──→ 输出检视结论
                │
                ↓
案由选择模块（本模块）
        │
        ├──→ 案由检索与分析
        ├──→ 案由确定
        └──→ 输出案由分析报告
```

配合要点：
- 必须先获取请求权分析模块的输出
- 案情特征标签用于检索关键词构造
- 主要规范用于案由匹配验证
