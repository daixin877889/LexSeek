# 案由检索法

从案情特征标签构造检索关键词，执行多轮检索《民事案件案由规定》的方法。

【重要提醒】本文档中的所有关键词示例仅为检索关键词，用于调用检索工具搜索《民事案件案由规定》。这些关键词不代表案由名称，也不保证检索结果存在。案由名称必须通过检索验证后方可确定，禁止将检索关键词直接当作案由输出。

## 检索关键词构造规则

### 固定结构

```
检索关键词 = 固定前缀 + 案情特征标签
```

- **固定前缀**：`民事案件案由规定`
- **后缀**：案情特征标签

### 案情特征标签类型与后缀构造

| 案情特征标签类型 | 检索关键词后缀示例（非案由名称） |
|------------------|--------------------------------|
| 用工形式特征（外卖骑手、网约车司机） | `新就业形态用工`、`用工纠纷`、`劳动关系` |
| 法律关系特征（租赁合同、建设工程） | `租赁合同`、`建设工程合同` |
| 行业领域特征（金融、保险） | `金融`、`保险` |
| 争议类型特征（确认劳动关系、损害赔偿） | `确认劳动关系`、`损害赔偿` |

### 构造示例

```markdown
案情特征标签：外卖骑手、劳动关系确认

检索关键词构造：
- `民事案件案由规定 新就业形态用工`
- `民事案件案由规定 用工纠纷`
- `民事案件案由规定 劳动关系`
```

## 多轮检索规则

当首轮检索结果不完整或不理想时，执行多轮检索：

### 第一轮：精确检索

**目标**：检索案情特征标签直接对应的案由

**关键词**：`民事案件案由规定 <案情特征标签精确表述>`

**判断标准**：
- 检索到专门案由 → 成功，进入版本校验
- 未检索到专门案由 → 进入第二轮

### 第二轮：扩大检索

**目标**：检索相邻领域或相关类型的案由

**关键词构造**：
- 扩大案情特征标签的范围
- 检索相关领域的案由

**示例**：
```markdown
首轮：`民事案件案由规定 新就业形态用工` → 无结果
第二轮：`民事案件案由规定 劳动合同纠纷` → 检索到候选案由
```

### 第三轮：通用检索

**目标**：检索通用案由作为备选

**关键词**：`民事案件案由规定 合同纠纷`、`民事案件案由规定 侵权责任纠纷`

**适用情形**：
- 第一轮、第二轮均无专门案由匹配
- 需要采用通用案由并说明原因

## 检索结果提取

从检索结果中提取以下信息：

| 信息项 | 说明 |
|--------|------|
| 案由名称 | 完整的案由表述 |
| 案由编号 | 如有编号，记录编号 |
| 生效日期 | 用于版本校验 |
| 适用范围 | 案由的适用情形说明 |

## 检索验证

最终确定的案由必须通过检索验证：

1. 使用 `search_law` 工具验证案由名称存在于《民事案件案由规定》
2. 案由名称必须严格匹配检索结果中的表述
3. 禁止自行编造或修改案由名称
