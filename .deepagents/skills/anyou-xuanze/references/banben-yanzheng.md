# 版本校验法

校验案由生效日期，判断版本是否为最新版的方法。

## 版本判定规则

### 生效日期判定标准

| 生效日期 | 版本判定 | 处理方式 |
|----------|----------|----------|
| ≥2026-01-01 | 最新版案由规定 | 可直接采用 |
| <2026-01-01 | 旧版案由 | 需确认是否已被新案由替代或仍有效 |

### 版本判定流程

```
检索结果输入
    │
    └──→ 检查生效日期字段
            │
            ├──≥2026-01-01 → 判定为最新版 → 可直接采用
            │
            └──<2026-01-01 → 判定为旧版 → 需进一步确认
                    │
                    ├── 检索是否有新版替代
                    │       │
                    │       ├── 有替代 → 采用新版
                    │       └── 无替代 → 确认旧版仍有效后采用
                    │
                    └── 标注版本说明
```

## 旧版案由处理

### 确认旧版是否仍有效

**确认方法**：
1. 检索最新版《民事案件案由规定》是否包含该案由
2. 检索是否有官方文件明确废止该案由

**处理原则**：
- 若新版仍包含该案由 → 采用新版表述
- 若新版已废止该案由 → 禁止采用，选择替代案由
- 若新版未提及但无废止文件 → 可谨慎采用并说明原因

### 版本说明格式

在输出中标注版本说明：

```markdown
版本说明：
- 案由名称：<案由>
- 生效日期：<日期>
- 版本判定：<最新版/旧版已确认有效>
```

## 新版案由优先原则

**适用情形**：同一纠纷类型存在新旧版本案由

**处理规则**：
- 新版案由优先于旧版案由
- 采用新版案由表述和编号
- 注明新版生效日期

## 版本校验记录

建议记录版本校验过程：

```markdown

- 版本判定：<最新版/旧版>
- 处理方式：<直接采用/确认后采用/选择替代>
- 最终采用：<案由名称>，版本：<版本说明>
```
