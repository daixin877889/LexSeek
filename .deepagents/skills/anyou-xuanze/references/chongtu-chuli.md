# 冲突处理法

案由选择过程中冲突情形的处理规则。

## 冲突情形类型

| 冲突情形 | 处理方式 |
|----------|----------|
| 检索到新设专门案由 vs 过往经验中的通用案由 | 采用检索到的专门案由 |
| 检索到多个专门案由候选 | 选择与主要规范最匹配的案由 |
| 检索结果与过往经验冲突 | 检索结果为准 |
| 检索无结果 | 扩大检索范围，若仍无结果则采用最相近的通用案由并说明原因 |

## 冲突处理原则

### 核心原则：检索优先

检索结果优先于任何过往经验、模板、内部建议。

```markdown
示例：
过往经验：劳动争议案件通常采用"劳动合同纠纷"
检索结果：2026版新增"新就业形态用工纠纷"

处理：采用检索到的"新就业形态用工纠纷"
```

### 去锚定原则

本案标题（如有）仅为行政归档代号，不具有任何法律定性效力。

必须在逻辑上完全屏蔽标题的语义干扰，仅依据案情特征标签与主要规范进行独立的案由选择。

## 冲突情形详解

### 一、检索结果 vs 过往经验

**情形**：检索到的案由与过往经验中的常用案由不同

**处理**：
- 检索结果为准
- 在输出中说明为何不采用过往经验中的案由

**说明格式**：
```markdown
过往经验案由：<案由名称>
检索结果案由：<案由名称>

选择理由：
- 检索结果为最新版案由规定中的专门案由
- 该案由直接匹配案情特征标签
- 该案由对应最新的规范体系
```

### 二、多个专门案由候选

**情形**：检索到多个专门案由同时匹配案情特征标签

**处理**：
- 选择与主要规范最匹配的案由
- 在输出中说明选择理由

**判断标准**：
- 案由对应的主要规范是否包含本案检视结论所依据的法条
- 案由适用范围是否更精确匹配本案纠纷类型

**说明格式**：
```markdown
候选专门案由：
1. <案由名称1> → 对应主要规范：<规范名称>
2. <案由名称2> → 对应主要规范：<规范名称>

选择理由：
- <案由名称1>对应的主要规范包含本案检视结论所依据的法条
- <案由名称1>适用范围更精确匹配本案纠纷类型
```

### 三、检索结果与案件标题冲突

**情形**：检索到的案由与案件标题暗示的案由不同

**处理**：
- 完全屏蔽标题语义干扰
- 仅依据案情特征标签与主要规范选择案由

**说明格式**：
```markdown
案件标题：<标题内容>

去锚定说明：
- 案件标题仅为行政归档代号，不具有法律定性效力
- 依据案情特征标签检索结果选择案由
- 最终案由：<案由名称>
```

### 四、检索无结果

**情形**：多轮检索均未找到匹配的专门案由

**处理**：
1. 扩大检索范围，检索相邻领域案由
2. 若仍无结果，采用最相近的通用案由
3. 必须说明采用通用案由的原因

**说明格式**：
```markdown
检索记录：
1. 第一轮：`民事案件案由规定 <关键词>` → 无结果
2. 第二轮：`民事案件案由规定 <关键词>` → 无结果
3. 第三轮：`民事案件案由规定 合同纠纷` → 检索到通用案由

采用通用案由原因：
- 经多轮检索，无专门案由匹配案情特征标签
- 采用最相近的通用案由"<案由名称>"
- 该通用案由适用范围涵盖本案纠纷类型
```

## 与请求权基础skill竞合处理的关联

本skill的冲突处理可参考 `qingqiuquan-jichu` skill中的竞合处理法：

| 本skill冲突处理 | 请求权skill竞合处理 |
|-----------------|---------------------|
| 检索结果优先于过往经验 | 规范检索结果优先于推测 |
| 多个专门案由竞合 → 选择最匹配 | 规范排斥竞合 → 选择特别规范 |
| 无专门案由 → 采用通用案由 | 无特别规范 → 采用一般规范 |
