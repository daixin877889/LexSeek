# 优先级规则

案由选择的优先级排序规则，确保专门案由优先于通用案由。

## 优先级层级

案由选择按以下优先级排序：

| 优先级 | 选择标准 | 说明 |
|--------|----------|------|
| **第一优先** | 专门案由 | 案由名称直接匹配案情特征标签 |
| **第二优先** | 新增案由 | 2026版新增的案由往往对应新规范或新纠纷类型 |
| **第三优先** | 通用案由 | 无专门案由匹配时采用，需说明原因 |

## 第一优先：专门案由

### 定义

专门案由是指案由名称直接反映案情特征标签的案由。

### 识别标准

案由名称中包含案情特征标签的核心要素：

```markdown
示例：
案情特征标签：建设工程合同纠纷
专门案由：建设工程合同纠纷（案由名称直接匹配）
```

### 采用条件

1. 案由名称与案情特征标签直接匹配
2. 案由适用范围涵盖本案纠纷类型
3. 版本为最新版或旧版已确认有效

## 第二优先：新增案由

### 定义

新增案由是指最新版《民事案件案由规定》中新设的案由。

### 识别标准

生效日期为2026-01-01及之后的新设案由。

### 采用条件

1. 无直接匹配的专门案由
2. 新增案由对应新规范或新纠纷类型
3. 新增案由适用范围涵盖本案纠纷类型

### 典型情形

```markdown
示例：
案情特征标签：新就业形态用工纠纷
新增案由：新就业形态用工纠纷（2026版新增）
```

## 第三优先：通用案由

### 定义

通用案由是指适用范围广泛、可涵盖多种纠纷类型的案由。

### 典型通用案由

- 合同纠纷
- 侵权责任纠纷
- 劳动争议

### 采用条件

1. 无专门案由匹配本案
2. 无新增案由匹配本案
3. 通用案由适用范围涵盖本案纠纷类型

### 必须说明原因

采用通用案由时，必须在输出中说明：

```markdown
采用通用案由原因：
- 经检索，无专门案由直接匹配案情特征标签
- 经检索，无新增案由匹配本案纠纷类型
- 通用案由"<案由名称>"适用范围涵盖本案纠纷
```

## 优先级选择流程

```
候选案由列表
    │
    ├── 检查是否有专门案由？
    │       │
    │       ├── 有 → 采用专门案由（第一优先）
    │       └── 无 → 进入下一优先级
    │
    ├── 检查是否有新增案由？
    │       │
    │       ├── 有 → 采用新增案由（第二优先）
    │       └── 无 → 进入下一优先级
    │
    ├── 检查是否有匹配的通用案由？
    │       │
    │       ├── 有 → 采用通用案由（第三优先）+ 说明原因
    │       └── 无 → 扩大检索范围
```

## 优先级冲突处理

当多个优先级层次的案由同时存在时：

| 冲突情形 | 处理方式 |
|----------|----------|
| 专门案由 vs 通用案由 | 采用专门案由 |
| 新增专门案由 vs 旧版专门案由 | 采用新增专门案由 |
| 多个专门案由候选 | 选择与主要规范最匹配的案由 |
