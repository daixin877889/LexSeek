## f01: 证据"三性"审查框架

```yaml
- id: f01
  title: 证据"三性"审查框架
  type: framework
  source_chapter: 第一章 · 证据辩护导论
  source_quote: |
    "辩护人可以通过质疑控方每一项证据的'两力'（证据能力、证明力）、'三性'（合法性、真实性、关联性），进而攻破证据锁链。"
  summary: |
    审查任何证据时，依次检验三项属性：
    1. 合法性：证据是否依法收集，有无非法取证
    2. 真实性：证据内容是否真实，有无伪造篡改
    3. 关联性：证据与案件事实是否有实质联系
    
    但不能拘泥于三性，需综合运用证据规则，质证与辩论融为一体。
  tags: [framework, evidence-review, foundational]
```

## f02: 实物证据鉴真框架

```yaml
- id: f02
  title: 实物证据鉴真框架
  type: framework
  source_chapter: 第二章 · 物证、书证
  source_quote: |
    "鉴真主要解决证据的同一性、载体真实性、形式关联性问题，否则不具备证据资格。"
  summary: |
    实物证据必须先鉴真才能进入质证程序。鉴真分两步：
    1. 独特性辨认：通过外部特征（形状、颜色、签名等）识别证据同一性
    2. 保管链验证：追踪证据从提取到出示的全流程，确保未被调包或污染
    
    鉴真解决的是"证据能力"问题，而非"证明力"问题。
  tags: [framework, physical-evidence, authentication]
```

## f03: 非法证据排除申请框架

```yaml
- id: f03
  title: 非法证据排除申请框架
  type: framework
  source_chapter: 第九章 · 非法证据排除
  source_quote: |
    "被告人和辩护人只需提供非法取证的材料或线索，而无须证明侦查机关取证违法。"
  summary: |
    申请排除非法证据的流程：
    1. 提出申请：提供线索或材料（不需证明非法）
    2. 说明理由：具体说明何种非法方法
    3. 启动调查：检察院承担证明合法性的责任
    4. 调查程序：先行当庭调查为原则
    5. 裁决结果：不能证明合法性则排除
    
    关键：证明责任在控方，辩方只需提供线索。
  tags: [framework, illegal-evidence, exclusion]
```

## f04: 鉴定意见质证框架

```yaml
- id: f04
  title: 鉴定意见质证框架
  type: framework
  source_chapter: 第五章 · 鉴定意见
  source_quote: |
    "鉴定意见是重要的客观证据，但也必须经过严格的审查。"
  summary: |
    质证鉴定意见的系统框架：
    1. 主体审查：鉴定机构/鉴定人是否有资质
    2. 检材审查：来源是否清楚，保管链是否完整
    3. 程序审查：是否依法告知，是否复核
    4. 方法审查：是否符合专业规范
    5. 内容审查：意见是否明确，是否超范围鉴定
    6. 关联审查：与案件事实有无关联，与其他证据是否矛盾
    
    任一环节出问题，鉴定意见不得作为定案根据。
  tags: [framework, expert-opinion, forensic]
```

## f05: 间接证据定案判断框架

```yaml
- id: f05
  title: 间接证据定案判断框架
  type: framework
  source_chapter: 第十章 · 证据的综合审查与运用
  source_quote: |
    "没有直接证据，但间接证据同时符合下列条件的，可以认定被告人有罪：（一）证据已经查证属实；（二）证据之间相互印证..."
  summary: |
    运用间接证据定案需同时满足五个条件：
    1. 证据已查证属实
    2. 证据之间相互印证，无矛盾或矛盾已排除
    3. 全案证据形成完整证据链
    4. 排除合理怀疑，结论具有唯一性
    5. 推理符合逻辑和经验
    
    五个条件缺一不可，死刑案件更需慎重。
  tags: [framework, indirect-evidence, burden-of-proof]
```

## f06: 口供补强判断框架

```yaml
- id: f06
  title: 口供补强判断框架
  type: framework
  source_chapter: 第十章 · 证据的综合审查与运用
  source_quote: |
    "根据被告人的供述、指认提取到了隐蔽性很强的物证、书证，且被告人的供述与其他证明犯罪事实发生的证据相互印证..."
  summary: |
    口供必须有其他证据补强才能定案。判断要点：
    1. 需要补强的是能直接证明主要事实的有罪供述（不含辩解）
    2. 补强证据必须是口供以外的其他证据
    3. 同案犯口供原则上不能互为补强
    4. "先供后证"提取的隐蔽物证有较好补强性
    
    只有口供没有其他证据，不能定罪。
  tags: [framework, confession, corroboration]
```

## f07: 证明责任分配框架

```yaml
- id: f07
  title: 证明责任分配框架
  type: framework
  source_chapter: 第十章 · 证据的综合审查与运用
  source_quote: |
    "公诉案件中被告人有罪的举证责任由人民检察院承担。"
  summary: |
    刑事诉讼证明责任分配规则：
    1. 基本规则：控方承担证明被告人有罪的责任
    2. 被告人不负证明责任（无罪推定）
    3. 例外情形：法律明确规定被告人承担（如巨额财产来源不明罪）
    4. 积极事实：主张积极事实的一方承担证明责任
    
    排除非法证据程序中，控方承担证明取证合法的责任。
  tags: [framework, burden-of-proof, presumption]
```

## f08: 证明标准判断框架

```yaml
- id: f08
  title: 证明标准判断框架
  type: framework
  source_chapter: 第十章 · 证据的综合审查与运用
  source_quote: |
    "证据确实、充分，应当符合以下条件：（一）定罪量刑的事实都有证据证明；（二）据以定案的证据均经法定程序查证属实；（三）综合全案证据，对所认定事实已排除合理怀疑。"
  summary: |
    刑事证明标准"证据确实、充分"的三要件：
    1. 定罪量刑事实都有证据证明（全面性）
    2. 定案证据均经法定程序查证属实（合法性和真实性）
    3. 综合全案排除合理怀疑，结论唯一（排他性）
    
    死刑案件标准更高：排除一切合理怀疑。
  tags: [framework, standard-of-proof, criminal]
```

## f09: 印证规则判断框架

```yaml
- id: f09
  title: 印证规则判断框架
  type: framework
  source_chapter: 第十章 · 证据的综合审查与运用
  source_quote: |
    "证据之间相互印证，就是两个以上证据所包含的事实信息得到了相互验证。"
  summary: |
    判断证据是否相互印证的方法：
    1. 寻找两个以上证据的事实信息重合或交叉点
    2. 检验重合信息能否相互验证真实性
    3. 注意：印证解决的是真实性，不解决合法性
    
    特殊情形：
    - 证人翻证：需有其他证据印证才能采信庭审证言
    - 被告人翻供：庭前供述有其他证据印证可采信
    - 证人有利害关系：需有其他证据印证才能采信
  tags: [framework, corroboration, verification]
```

## f10: 孤证识别框架

```yaml
- id: f10
  title: 孤证识别框架
  type: framework
  source_chapter: 第十章 · 证据的综合审查与运用
  source_quote: |
    "孤证不能定案，即单个证据不能单独作为认定案件事实的根据。"
  summary: |
    判断是否为孤证的方法：
    1. 形式判断：只有一个证据形式上呈现
    2. 实质判断：虽有多份证据，但与主要事实有关联的只有一项
    3. 关联判断：其他证据与主要事实无实质关联
    
    孤证规则仅适用于对被告人不利的事实认定。
    常见孤证情形：只有证人指证、只有被告人供述。
  tags: [framework, single-evidence, prohibition]
```

## f11: "一对一"证据处理框架

```yaml
- id: f11
  title: "一对一"证据处理框架
  type: framework
  source_chapter: 第十章 · 证据的综合审查与运用
  source_quote: |
    "在证据形成'一对一'的情形下...一方言词证据为真的概率只有50%，连优势证据的证明标准都达不到。"
  summary: |
    处理"一对一"证据（如行贿人指控、被告人否认）的方法：
    1. 本质判断：实质上属于孤证
    2. 标准判断：连优势证据标准（51%）都达不到
    3. 结论判断：存疑有利于被告人，直接认定该项事实不存在
    
    适用情形：行贿受贿、强奸、猥亵、盗窃等私密空间案件。
  tags: [framework, one-to-one, doubt]
```

## f12: 辨认笔录审查框架

```yaml
- id: f12
  title: 辨认笔录审查框架
  type: framework
  source_chapter: 第六章 · 勘验、检查、辨认、侦查实验等笔录
  source_quote: |
    "辨认笔录具有下列情形之一的，不得作为定案的根据：（一）辨认不是在侦查人员主持下进行的；（二）辨认前使辨认人见到辨认对象的..."
  summary: |
    审查辨认笔录的六个排除情形：
    1. 非侦查人员主持
    2. 辨认前见到辨认对象
    3. 辨认没有个别进行
    4. 辨认对象未混杂或数量不符规定
    5. 有明显暗示或指认嫌疑
    6. 其他无法确定真实性的情形
    
    人辨认≥7人，照片辨认≥10张；物品辨认≥5件。
  tags: [framework, identification, procedure]
```

## f13: 勘验检查笔录审查框架

```yaml
- id: f13
  title: 勘验检查笔录审查框架
  type: framework
  source_chapter: 第六章 · 勘验、检查、辨认、侦查实验等笔录
  source_quote: |
    "对勘验、检查笔录应当着重审查以下内容：（一）勘验、检查是否依法进行..."
  summary: |
    审查勘验检查笔录的系统要点：
    1. 主体：是否侦查人员，是否持证，是否≥二人
    2. 见证人：是否与案件无关，是否适格
    3. 时间：是否及时进行
    4. 内容：是否现场制作，是否全面记载
    5. 签名：相关人员是否签名
    6. 附件：现场图、照片、录像是否相符
    
    明显不符合法律规定且无法解释的，不得作为定案根据。
  tags: [framework, inspection, on-site]
```

## f14: 同步录音录像运用框架

```yaml
- id: f14
  title: 同步录音录像运用框架
  type: framework
  source_chapter: 第九章 · 非法证据排除
  source_quote: |
    "讯问笔录记载的内容与讯问录音录像存在实质性差异的，以讯问录音录像为准。"
  summary: |
    运用同步录音录像的方法：
    1. 申请调取：证明取证合法性或供述真实性
    2. 详细观看：形成文字稿，对比笔录
    3. 发现差异：笔录与录像不一致时以录像为准
    4. 揭露非法：疲劳审讯、威胁、诱导等
    
    录像是最优证据，比笔录更完整、真实、客观。
    检方不提供录像，推定对控方不利。
  tags: [framework, recording, interrogation]
```

## f15: 证人证言真实性审查框架

```yaml
- id: f15
  title: 证人证言真实性审查框架
  type: framework
  source_chapter: 第三章 · 证人证言、被害人陈述
  source_quote: |
    "对证人证言应当着重审查...证言之间以及与其他证据之间能否相互印证，有无矛盾。"
  summary: |
    审查证人证言真实性的四个维度：
    1. 利害关系：证人与当事人有无利害关系
    2. 证言矛盾：前后矛盾或与其他证据矛盾
    3. 印证情况：能否与其他证据相互印证
    4. 常理判断：是否符合常识、常理
    
    关键证人应申请出庭接受询问。
  tags: [framework, witness, credibility]
```
