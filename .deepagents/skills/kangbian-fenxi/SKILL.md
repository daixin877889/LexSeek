# 抗辩策略分析方法论

## 核心原则

抗辩策略分析以"攻防博弈"为核心视角，在已确认请求权基础的前提下，系统挖掘事实风险、构建抗辩策略、预判反制方案。

分析方法遵循三个基本原则：

1. **合同解释优先原则**：先探究当事人意思表示（条款含义、交易目的），再进行法律评价
2. **效力体系分层原则**：成立（事实判断）→有效（价值判断）→生效（履行效力限制）三阶段递进检视
3. **普遍性验证原则**：抗辩策略须具有普适性，仅适用于本案特殊情形的策略可能不被接受

## 输入校验与自定位机制

### 检验输入来源

**场景A：顺序调用（有前置模块）**
- 输入包含「已确认适用的请求权基础」标记
- 输入包含「已确定的案由」标记
- 输入包含「已排除的请求权基础及理由」
- 直接承接上下文，进入抗辩分析

**场景B：单独调用（无前置模块）**
- 输入缺失前置标记
- 执行「自定位流程」

### 自定位流程（单独调用时执行）

**目标**：快速定位请求权基础，让抗辩分析能够进行

**步骤**：

1. **诉讼请求识别**
   - 从案件材料中提取原告主张的诉讼请求
   - 识别请求内容：给付、返还、赔偿、确认等
   - 识别请求对象：具体金额、特定物、行为等

2. **请求权类型定位**
   - 根据诉讼请求快速定位可能的请求权类型
   - 例：「支付货款」→ 合同请求权 → 买卖合同
   - 例：「返还财产」→ 物上请求权或不当得利
   - 例：「赔偿损失」→ 侵权请求权或违约责任

3. **案由初步确定**
   - 根据请求权类型确定可能的案由类别
   - 例：合同请求权 + 买卖关系 → 买卖合同纠纷
   - 例：物上请求权 + 不动产 → 物权纠纷

4. **进入抗辩分析**
   - 基于定位结果执行抗辩策略分析
   - 针对定位的请求权基础构建抗辩

**与请求权基础模块的区别**：

| 维度 | 抗辩模块自定位 | 请求权基础模块 |
|------|----------------|----------------|
| 目标 | 功能性定位（让抗辩能运行） | 专业性论证 |
| 方法 | 快速识别诉讼请求→定位请求权类型 | 完整三层四步法检视 |
| 深度 | 不做精细规范分类、举证分配 | 规范分类、举证分配、竞合处理 |
| 输出 | 定位结果直接用于抗辩 | 完整请求权基础论证报告 |

### 输出标注规则

**顺序调用时**：
- 输出报告标注「基于已确认的请求权基础分析」
- 不提示额外模块

**单独调用时**：
- 输出报告标注「本次分析基于抗辩模块内部快速定位」
- 输出报告末尾提示：「若需更精细的请求权基础论证，可单独运行请求权基础分析模块」

## 关键要点

### 一、合同效力体系检视
详见 [references/合同效力检视.md](./references/合同效力检视.md)

**核心区分**：
- **成立**：事实判断（有没有），产生形式拘束力（不得擅自变更解除）
- **有效**：价值判断（好不好），产生实质拘束力（请求履行、保有给付）
- **生效**：效力限制（能否行使），涉及履行效力（可被条件限制）

**检视要点**：
- 成立阶段：确认合意是否存在，主体、意思表示、合意
- 有效阶段：推定有效、质疑者举证，行为能力、意思真实、不违法背俗
- 生效阶段：检验条件/期限/批准限制，条件限制特定义务而非整体

### 二、附条件合同分析
详见 [references/附条件分析.md](./references/附条件分析.md)

- 识别合同的全部义务
- 判断哪个义务附了条件
- 分析条件对各义务的独立影响
- 核心：条件限制的是"特定义务的履行效力"，非"整个合同"

### 三、条件不成就→解除条件成就路径
详见 [references/条件不成就解除.md](./references/条件不成就解除.md)

**高级技巧**：生效条件不成就，可解释为解除条件成就
- 157条（无效后果）：缔约过失责任
- 566条（解除后果）：违约责任
- 关键：从交易背景解释"条件不成就则合同消灭"的效果意思

### 四、批准生效合同分析
详见 [references/批准生效合同.md](./references/批准生效合同.md)

**核心规则**：批准生效≠整个合同不生效
- 只有权利变动条款需批准
- 其他条款可独立生效
- 违反独立生效条款构成违约（而非缔约过失）

### 五、规范目的导向解释
详见 [references/规范目的解释.md](./references/规范目的解释.md)

**核心原则**：以规范目的为导向判断管制范围
- 识别批准规范的规范目的
- 匹配条款功能与目的关系
- 最小范围原则：将管制限制在最小范围

### 六、反射式迭代检索
详见 [references/反射式检索.md](./references/反射式检索.md)

- 周期一：宽泛检索，识别核心主体与争议标的
- 周期二：二次扫描，提炼专业关键词
- 周期三：精准定位，关联司法解释

### 七、攻防策略推演（三分法分类）
详见 [references/攻防策略.md](./references/攻防策略.md)

按请求权检视阶段将抗辩分为三类，逐类检视：
- **权利妨碍抗辩**：请求权根本没有产生（如无行为能力、意思表示瑕疵、合同未成立）
- **权利消灭抗辩**：请求权已产生但已消灭（如清偿、免除、抵销、合同已解除）
- **权利限制抗辩**：请求权存在但暂不能行使（如时效抗辩、同时履行抗辩权、先履行抗辩权）

推演结构：
- 被告视角：事实风险 + 法律依据 = 抗辩理由
- 原告视角：针对抗辩的反制策略
- 普遍性验证：策略是否具有普适性

### 八、合同解释优先
详见 [references/合同解释优先.md](./references/合同解释优先.md)

- 探究当事人真实意思表示
- 条款含义解释优先于法律评价
- 交易目的作为解释辅助

### 九、普遍性验证
详见 [references/普遍性验证.md](./references/普遍性验证.md)

**核心论证原则**：方案必须有普适性，强调"本案特殊"不会被接受

- 同类案件验证：若本案这样处理，同类案件是否也应如此
- 制度一致性检验：策略是否符合制度目的
- 反例测试：是否存在明显不合理的推论
- 极端问题验证：将问题推到极端，验证规则是否仍成立

### 十、自定位流程（单独调用时执行）
详见 [references/自定位流程.md](./references/自定位流程.md)

**触发条件**：用户单独调用抗辩模块（未先运行请求权基础和案由模块）

**定位性质**：功能性定位（让分析能运行），而非专业性论证

**定位步骤**：
1. 诉讼请求识别 → 从案件材料提取诉讼请求
2. 请求权类型定位 → 根据请求内容匹配请求权类型
3. 案由初步确定 → 根据请求权类型确定案由

### 十一、输出格式规范
详见 [references/报告输出格式.md](./references/报告输出格式.md)

**报告结构**：
- 报告开头（标注分析模式）
- 案件概况
- 请求权基础确认
- 合同效力检视（如涉及）
- 抗辩策略推演（被告→原告→验证）
- 策略建议
- 报告末尾（标注来源，单独调用时提示）

**抗辩效力标注**：每项抗辩必须标注效力（强/中等/弱）

## 与工具配合

本 skill 需配合 search_law 工具使用：

| 工具 | 调用时机 | 用途 |
|------|----------|------|
| search_law | 法律检索阶段 | 检索支撑抗辩/反驳的法条、司法解释 |

**调用方式**：
1. 完成事实风险挖掘和合同效力检视后
2. 根据抗辩点提炼法律关键词
3. 调用 search_law 进行检索
4. 检索结果用于支撑抗辩内容和原告反驳的法律依据

**检索关键词来源**：
- 抗辩类型（如"不可抗力""违约金过高""人格混同"）
- 案件争议焦点（如"合同解除""股东责任"）
- 具体法条需求（如"民法典相关条文""公司法相关条文"）

详见 [references/反射式检索.md](./references/反射式检索.md)

## 使用流程

### 场景A：顺序调用流程（有前置模块）

1. 输入校验 → 确认已传递「已确认适用的请求权基础」
2. 承接上下文 → 直接使用已确定的请求权基础和案由
3. 事实风险挖掘 → 运用合同解释优先原则审查案件材料
4. 合同效力检视 → 如涉及合同，按成立→有效→生效三阶段检视
5. 附条件分析 → 如涉及附条件合同，精细化分析义务与条件关系
6. 条件不成就处理 → 如生效条件不成就，判断是否可解释为解除条件成就
7. 批准生效分析 → 如涉及批准合同，判断部分条款是否独立生效
8. 法律检索 → 调用 search_law 工具，检索支撑抗辩/反驳的法条
9. 攻防推演 → 构建被告抗辩策略与原告反驳
10. 普遍性验证 → 提出普遍性问题，验证策略的普适性
11. 报告输出 → 标注「基于已确认的请求权基础分析」

### 场景B：单独调用流程（无前置模块）

1. 输入校验 → 检测缺失前置标记，触发自定位流程
2. 诉讼请求识别 → 从案件材料提取原告主张的诉讼请求
3. 请求权类型定位 → 根据诉讼请求快速定位可能的请求权类型
4. 案由初步确定 → 根据请求权类型确定可能的案由
5. 事实风险挖掘 → 运用合同解释优先原则审查案件材料
6. 合同效力检视 → 如涉及合同，按成立→有效→生效三阶段检视
7. 附条件分析 → 如涉及附条件合同，精细化分析义务与条件关系
8. 条件不成就处理 → 如生效条件不成就，判断是否可解释为解除条件成就
9. 批准生效分析 → 如涉及批准合同，判断部分条款是否独立生效
10. 法律检索 → 调用 search_law 工具，检索支撑抗辩/反驳的法条
11. 攻防推演 → 构建被告抗辩策略与原告反驳
12. 普遍性验证 → 提出普遍性问题，验证策略的普适性
13. 报告输出 → 标注「本次分析基于抗辩模块内部快速定位」，提示可单独运行请求权基础模块
