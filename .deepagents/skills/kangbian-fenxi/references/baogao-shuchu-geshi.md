# 抗辩分析报告输出格式

抗辩分析报告应按以下标准结构输出。

## 重要规则

**强制要求**：
- **报告以 `# 抗辩分析及应对策略预测` 开头**，标题前不得有任何内容

**禁止事项**：
- 报告中不得出现任何表情符号
- 抗辩效力用文字标注（强/中等/弱），不用符号

## 报告结构

### 一、报告标题与开头

**标题**：
```
# 抗辩分析及应对策略预测
```

**顺序调用时**：
标题后直接开始事实风险挖掘，不展示案件概要和请求权基础定位。

**单独调用时**：
标题后用一句告知：
```
因您前面未选择请求权基础分析及案由分析模块，本次分析将基于诉讼请求识别进行快速定位和分析。
```

然后可简要展示定位结果：
```
定位情况：
- 请求权类型：[定位的类型]
- 案由：[确定的案由]
```

### 二、案件概况（仅单独调用时展示）

简要陈述案件背景：
- 当事人关系
- 争议焦点
- 核心诉求

**顺序调用时跳过此部分**。

### 三、请求权基础定位（仅单独调用时展示）

```
定位情况：
- 诉讼请求：[提取的诉讼请求内容]
- 请求权类型：[定位的类型]
- 可能的请求权基础：[匹配的可能条文]
- 案由：[确定的案由]
```

**顺序调用时跳过此部分，直接从事实风险挖掘开始**。

### 四、事实风险挖掘

针对案件材料的事实风险点进行挖掘，使用表格展示：

| 风险点编号 | 风险类型 | 具体内容 | 影响分析 |
|------------|----------|----------|----------|
| 1 | [类型] | [内容] | [影响] |
| 2 | [类型] | [内容] | [影响] |

### 五、合同效力检视（如涉及合同）

使用表格展示：

| 检视阶段 | 判断性质 | 判断内容 | 检视结果 | 分析说明 |
|----------|----------|----------|----------|----------|
| 成立检视 | 事实判断 | 主体、意思表示、合意 | [成立/不成立] | [具体分析] |
| 有效检视 | 价值判断 | 行为能力、意思真实、不违法背俗 | [有效/无效/可撤销/效力待定] | [如有瑕疵，说明瑕疵类型] |
| 生效检视 | 效力限制 | 条件/期限/批准 | [已生效/未生效/部分生效] | [限制情况说明] |

### 六、抗辩策略推演（合并展示格式）

按抗辩点逐一展开，每个抗辩点包含：抗辩内容、反制策略、普遍性验证。

**格式模板**：

---

**抗辩一：[抗辩名称]**

抗辩内容：
- 事实锚点：[支撑抗辩的案件事实]
- 法律依据：[支撑抗辩的法条]
- 论证逻辑：[连接事实与法律的推理]
- 抗辩效力：[强/中等/弱]

原告反驳：
- 反驳内容：[针对抗辩的应对主张]
- 法律依据：[对抗法条]
- 事实补救空间：[可补充的证据或解释]

普遍性验证：
- 同类案件推演：[若本案这样处理，同类案件是否也应如此]
- 制度目的检验：[是否符合制度设计目的]
- 验证结论：[通过/部分通过/未通过]

---

**抗辩二：[抗辩名称]**

...（同上结构）

---

### 七、附条件分析（如涉及）

使用表格展示：

| 义务内容 | 是否附条件 | 条件内容 | 条件状态 | 效力状态 |
|----------|------------|----------|----------|----------|
| [义务A] | [是/否] | [如附条件] | [成就/未成就] | [已生效/暂不履行] |

### 八、批准生效分析（如涉及）

使用表格展示：

| 条款类型 | 与规范目的关系 | 效力状态 | 分析说明 |
|----------|----------------|----------|----------|
| [需批准条款] | 冲突 | [待批准/已批准] | [权利变动条款需批准] |
| [条款A] | 一致 | 可独立生效 | [不涉及权利变动] |

### 九、策略建议

使用表格展示：

**被告建议**

| 项目 | 内容 |
|------|------|
| 优先策略 | [推荐优先采取的抗辩] |
| 次要策略 | [可备选的抗辩] |
| 风险提示 | [需注意的风险点] |
| 证据建议 | [需要补充的证据] |

**原告建议**（如用户为原告）

| 项目 | 内容 |
|------|------|
| 应对要点 | [针对被告抗辩的应对方案] |
| 证据补强 | [需要准备的证据] |
| 预判风险 | [可能面临的败诉风险] |

### 十、报告末尾

**顺序调用时**：
无需额外提示。

**单独调用时**：
```
提示：若需更精细的请求权基础论证（包括三层四步法检视、规范分类、举证分配、竞合处理），建议先运行请求权基础分析模块和案由分析模块。
```

## 抗辩效力标准

| 抗辩效力 | 定义 |
|----------|------|
| 强 | 抗辩有较充分依据，可能被法院采纳 |
| 中等 | 抗辩有一定依据，需进一步完善 |
| 弱 | 抗辩依据不足，难以被法院采纳 |

---

## 输出格式要求

1. **标题强制**：报告以 `# 抗辩分析及应对策略预测` 开头，标题前不得有任何内容
2. **抗辩合并展示**：每个抗辩点下方直接展示原告反驳和普遍性验证，避免阅读跳跃
3. **抗辩效力标注**：使用"强/中等/弱"文字标注
4. **验证完整**：每项抗辩须通过普遍性验证
5. **内容区分**：顺序调用跳过案件概况和请求权基础定位
6. **表格使用**：事实风险、合同效力、附条件分析、批准生效分析使用表格；抗辩策略使用合并文字格式
