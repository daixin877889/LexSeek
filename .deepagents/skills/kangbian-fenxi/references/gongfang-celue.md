# 攻防策略推演

抗辩策略推演采用"被告-原告-普遍性"三视角结构。

## 推演结构

```
被告视角（抗辩策略）
    │
    └──→ 原告视角（反制预判）
              │
              └──→ 普遍性验证（策略检验）
```

## 第一视角：被告抗辩策略

**核心公式**：事实风险 + 法律依据 = 抗辩理由

### 事实风险点挖掘

从案件材料中识别对原告不利的事实：
- 前后矛盾之处
- 证据缺失环节
- 程序瑕疵问题
- 时效性问题

### 抗辩构建要素

| 要素 | 内容 | 来源 |
|------|------|------|
| 事实锚点 | 具体案件事实 | 案件材料 |
| 法律支撑 | 支撑抗辩的法条 | 反射式检索 |
| 论证逻辑 | 连接事实与法律 | 法律推理 |

### 抗辩类型分类

按请求权检视阶段分类：

| 抗辩类型 | 对应阶段 | 举例 |
|----------|----------|------|
| 权利未发生抗辩 | 步骤2 | 无行为能力、意思表示瑕疵 |
| 权利消灭抗辩 | 步骤3 | 清偿、免除、抵销 |
| 权利行使抗辩 | 步骤4 | 时效抗辩、同时履行抗辩权 |

## 第二视角：原告反制预判

**目标**：预判原告如何应对被告抗辩

### 反制策略类型

| 类型 | 内容 | 适用场景 |
|------|------|----------|
| 事实补救 | 解释不利事实、补充证据 | 事实争议 |
| 法律反击 | 引用对抗法条 | 法律争议 |
| 策略调整 | 变更诉讼请求、调整主张 | 策略层面 |

### 反制分析框架

针对被告的每项抗辩：
1. 分析抗辩的法律依据
2. 识别可能的反制法条
3. 评估事实补救空间
4. 判断风险等级

### 风险提示

为原告提供败诉风险评估：
- 高风险：抗辩可能成立，需重点应对
- 中风险：抗辩有一定依据，需准备应对
- 低风险：抗辩依据不足，但仍需注意

## 第三视角：普遍性验证

**核心问题**：若本案这样处理，同类案件是否也应如此处理？

### 验证方法

1. **同类案件推演**
   - 假设存在相同事实的案件
   - 适用相同策略
   - 检验结论是否合理

2. **制度目的检验**
   - 检验策略是否符合制度设计目的
   - 是否存在规避制度的嫌疑

3. **反例测试**
   - 构造极端案例
   - 检验是否存在不合理推论

### 验证结论

| 结论 | 说明 | 策略调整 |
|------|------|----------|
| 通过 | 策略具有普适性 | 维持策略 |
| 部分通过 | 策略需限定条件 | 限定适用范围 |
| 未通过 | 策略仅适用于本案特殊情形 | 重新构建策略 |

## 与其他方法的关系

攻防策略推演与其他方法的关系：

```
合同解释优先 → 挖掘事实风险点
       │
       └──→ 合同效力检视 → 确定抗辩类型
                 │
                 └──→ 反射式检索 → 提供法律支撑
                           │
                           └──→ 攻防策略推演
                                     │
                                     └──→ 普遍性验证
```

## 执行步骤

1. **事实风险挖掘**：运用合同解释优先原则审查案件材料
2. **抗辩策略构建**：被告视角构建抗辩主张
3. **法律支撑定位**：运用反射式检索定位法条
4. **反制策略预判**：原告视角预判反制方案
5. **普遍性验证**：检验策略普适性
6. **风险等级评估**：为每项策略标注风险等级
7. **策略优化**：根据验证结果调整策略

## 注意事项

1. **不预设立场**：客观分析双方优势劣势
2. **全面覆盖**：不遗漏可能的抗辩角度
3. **逻辑闭环**：每项策略需有事实锚点和法律支撑
4. **诚实评估**：如实评估风险等级，不夸大不隐瞒
5. **可操作性**：策略需具体可行，有明确行动指引
