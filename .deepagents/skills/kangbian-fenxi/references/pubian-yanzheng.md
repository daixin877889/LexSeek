# 普遍性验证

抗辩策略须具有普适性，仅适用于本案特殊情形的策略可能不被接受。

## 核心论证原则

**核心问题**：如果本案这样处理，同类案件是否也应这样处理？

法官/仲裁员不会接受"本案特殊所以要特别处理"的论证。只有普适规则才能被接受。

**失败模式**：
- 强调"本案特殊" → 法官不会为个案曲解规则
- 忽略同类案件 → 被攻击"还有其他案件你怎么处理"

## 验证原理

**核心思想**：法律适用应具有一致性，同类案件应同类处理。

```
本案策略 → 同类案件适用 → 结论是否合理 → 策略是否成立
```

## 验证维度

### 维度一：同类案件推演

**方法**：假设存在相同事实的案件，适用相同策略，检验结论合理性

**操作步骤**：
1. 提取本案策略的核心要素
2. 构造相同事实的假设案件
3. 适用相同策略
4. 检验结论是否合理

**判断标准**：
- 结论合理 → 策略通过
- 结论不合理 → 策略需修正
- 结论荒谬 → 策略不成立

### 维度二：制度目的检验

**方法**：检验策略是否符合制度设计目的

**操作步骤**：
1. 识别相关制度的立法目的
2. 检验策略适用是否偏离制度目的
3. 判断是否存在规避制度的嫌疑

**常见制度目的**：
| 制度 | 目的 | 策略检验 |
|------|------|----------|
| 诉讼时效 | 督促权利行使、维护秩序稳定 | 是否通过时效规避义务 |
| 不当得利 | 调整无法律原因的利益变动 | 是否滥用不当得利制度 |
| 撤销权 | 保护意思表示不真实方 | 是否滥用撤销权 |

### 维度三：反例测试

**方法**：构造极端情形，检验是否存在不合理推论

**操作步骤**：
1. 将策略推至极端
2. 构造极端假设情形
3. 检验结论是否荒谬
4. 判断策略边界

### 维度四：极端问题验证（高级技巧）

**方法**：将问题推到极端，验证规则是否仍成立

**核心洞察**：只有对极端问题得出肯定结论，诉讼方案才能被接受

**操作步骤**：
1. 提出极端假设情形
2. 适用相同策略
3. 判断结论是否仍合理
4. 若极端情形能接受，方案才稳健

## 验证结果处理

### 策略通过验证

**特征**：
- 同类案件适用合理
- 符合制度目的
- 无荒谬推论

**处理**：维持策略，可放心使用

### 策略部分通过

**特征**：
- 在一定条件下适用合理
- 需限定适用范围
- 存在边界情况

**处理**：明确策略适用条件，限定使用范围

### 策略未通过验证

**特征**：
- 同类案件适用产生不合理结果
- 偏离制度目的
- 存在荒谬推论

**处理**：重新构建策略，或放弃该策略

## 与攻防策略的关系

普遍性验证是攻防策略推演的最后环节：

```
攻防策略推演
    │
    ├──→ 被告抗辩策略
    │
    ├──→ 原告反制预判
    │
    └──→ 普遍性验证 ← 最后环节
```

## 执行步骤

1. **提取策略核心**：明确策略的核心主张和逻辑
2. **同类案件推演**：构造假设案件检验
3. **制度目的检验**：对照制度目的审查
4. **反例测试**：构造极端情形检验
5. **形成验证结论**：通过/部分通过/未通过
6. **策略调整**：根据验证结果调整策略

## 注意事项

1. **不预设结论**：客观验证，不为通过而通过
2. **诚实面对问题**：发现策略缺陷及时调整
3. **记录验证过程**：验证过程应可追溯
4. **区分策略与事实**：普遍性验证针对策略，非针对事实
5. **保持开放心态**：策略可调整，不必固守
