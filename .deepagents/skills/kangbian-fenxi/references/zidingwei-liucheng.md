# 自定位流程（单独调用时执行）

当用户单独调用抗辩模块（未先运行请求权基础和案由模块）时，执行自定位流程。

## 目标

快速定位请求权基础，让抗辩分析能够进行。

**定位性质**：功能性定位（让分析能运行），而非专业性论证（完整方法论）

## 流程步骤

### 步骤一：诉讼请求识别

**目标**：从案件材料中提取原告主张的诉讼请求

**操作要点**：

1. **识别起诉状/诉讼请求部分**
   - 查找案件材料中的"诉讼请求""起诉请求""请求事项"等标题
   - 若无明确标题，从案件叙述中推断

2. **识别请求内容**
   
   | 请求类型 | 特征 | 示例 |
   |----------|------|------|
   | 给付请求 | 要求对方支付/交付 | "请求被告支付货款100万元" |
   | 返还请求 | 要求对方归还 | "请求被告返还借款50万元" |
   | 赔偿请求 | 要求对方赔偿损失 | "请求被告赔偿损失20万元" |
   | 确认请求 | 要求确认法律关系 | "请求确认合同无效" |
   | 形成请求 | 要求变更/解除 | "请求解除合同" |

3. **识别请求对象**
   - 具体金额（如100万元）
   - 特定物（如某房产）
   - 特定行为（如办理过户）
   - 确认/变更法律关系

4. **输出定位结果**
   - 诉讼请求内容：[具体请求]
   - 请求类型：[给付/返还/赔偿/确认/形成]
   - 请求对象：[金额/物/行为/法律关系]

### 步骤二：请求权类型定位

**目标**：根据诉讼请求快速定位可能的请求权类型

**快速定位规则**：

| 诉讼请求特征 | 可能的请求权类型 | 典型案由 |
|--------------|------------------|----------|
| "支付货款""支付价款" | 合同请求权（买卖） | 买卖合同纠纷 |
| "支付租金" | 合同请求权（租赁） | 租赁合同纠纷 |
| "支付借款""偿还借款" | 合同请求权（借款） | 借款合同纠纷 |
| "返还财产""归还物品" | 物上请求权或不当得利 | 物权纠纷/不当得利纠纷 |
| "赔偿损失"（无合同背景） | 侵权请求权 | 侵权责任纠纷 |
| "赔偿损失"（有合同背景） | 违约责任 | 合同纠纷 |
| "确认合同无效" | 确认之诉 | 合同效力纠纷 |
| "解除合同" | 形成之诉 | 合同纠纷 |

**定位原则**：
- 有合同关系的优先定位为合同请求权
- 合同关系不明确的，需进一步审查案件材料
- 多种请求权可能并存时，选择最可能的类型

### 步骤三：案由初步确定

**目标**：根据请求权类型确定可能的案由类别

**案由定位规则**：

| 请求权类型 | 案由类别 | 典型案由 |
|------------|----------|----------|
| 合同请求权（买卖） | 合同纠纷 | 买卖合同纠纷 |
| 合同请求权（借款） | 合同纠纷 | 借款合同纠纷 |
| 合同请求权（租赁） | 合同纠纷 | 租赁合同纠纷 |
| 物上请求权 | 物权纠纷 | 排除妨害纠纷、返还原物纠纷 |
| 不当得利 | 不当得利纠纷 | 不当得利纠纷 |
| 侵权请求权 | 侵权责任纠纷 | 侵权责任纠纷 |

**案由确定原则**：
- 根据案件材料中提及的法律关系类型判断
- 若案件材料明确提及合同类型，直接定位对应案由
- 若案件材料未明确，根据当事人关系推断

### 步骤四：进入抗辩分析

**目标**：基于定位结果执行抗辩策略分析

**衔接要点**：
- 定位结果作为抗辩分析的输入
- 针对定位的请求权基础构建抗辩
- 不再回溯验证定位的正确性（时间效率优先）

---

## 与请求权基础模块的区别

| 维度 | 抗辩模块自定位 | 请求权基础模块 |
|------|----------------|----------------|
| **目标** | 功能性定位（让抗辩能运行） | 专业性论证 |
| **方法** | 快速识别→类型匹配 | 完整三层四步法检视 |
| **深度** | 识别请求权类型，不做精细分析 | 规范分类、举证分配、竞合处理 |
| **时间** | 快速定位，优先效率 | 完整检视，优先专业 |
| **输出** | 定位结果直接用于抗辩 | 完整请求权基础论证报告 |

**核心区分**：
- 抗辩模块自定位不做"三层四步法"的完整检视
- 抗辩模块自定位不做"规范分类法"的精细分析
- 抗辩模块自定位不做"举证分配法"的责任分配
- 抗辩模块自定位不做"竞合处理法"的多规范处理

用户若需要以上精细方法论分析，应单独运行请求权基础模块。

---

## 输出标注要求

当执行自定位流程时，抗辩报告必须标注：

### 报告开头标注

```
⚠️ 本次分析基于抗辩模块内部快速定位
定位结果：[请求权类型] → [案由]
定位方式：诉讼请求识别 → 类型匹配 → 案由确定
```

### 报告末尾提示

```
💡 提示：若需更精细的请求权基础论证（包括三层四步法检视、
规范分类、举证分配、竞合处理），可单独运行「请求权基础分析」模块。
```

---

## 定位失败处理

若诉讼请求无法识别或无法定位请求权类型：

1. **输出提示**
   - "无法从案件材料中识别明确的诉讼请求"
   - "无法定位可能的请求权基础类型"

2. **建议操作**
   - "建议先上传包含明确诉讼请求的案件材料"
   - "或先运行请求权基础分析模块"

3. **降级执行**
   - 基于已有案件材料做有限分析
   - 明确标注"定位不完全，分析范围受限"
