# 抗辩分析报告输出格式

抗辩分析报告应按以下标准结构输出。

## 重要规则

**强制要求**：
- **报告以 `# 案件抗辩分析报告` 开头**，标题前不得有任何内容

**禁止事项**：
- 报告中不得出现任何表情符号
- 抗辩效力用文字标注（强效力/中等效力/弱效力），不用符号

## 报告结构

### 一、分析范围与输入确认

**顺序调用时**：
简要引用已确认的请求权基础名称，不重复列明详情：
```
**本次分析针对的请求权基础：**
- [请求权基础1]：[简述确认理由]
- [请求权基础2]：[简述确认理由]

**已排除的请求权基础（不在本报告中详析）：**
- [被排除的请求权基础]：[排除理由摘要，一句话]
```

**单独调用时**：
标注"本次分析基于抗辩模块内部快速定位"，完整输出定位结果。

## 串联/独立模式规则

### 全局串联模式检测

输出前检查上下文中是否已存在"# 案件抗辩分析报告"或"# 请求权基础分析报告"输出。

- **串联模式**（检测到前置模块输出）：
  - 分析范围确认仅简要引用，不重复列明详情
  - 不输出案件概况
- **独立模式**（未检测到前置输出）：
  - 执行自定位流程后完整输出
  - 报告开头标注"本次分析基于抗辩模块内部快速定位"
  - 报告末尾提示运行前置模块

### 单独调用标注

**报告开头**：
```
因您前面未选择请求权基础分析及案由分析模块，本次分析将基于诉讼请求识别进行快速定位和分析。
```

**报告末尾**：
```
提示：若需更精细的请求权基础论证（包括三层四步法检视、规范分类、举证分配、竞合处理），建议先运行请求权基础分析模块和案由分析模块。
```

## 抗辩效力标准

| 抗辩效力 | 定义 | 典型情形 |
|----------|------|----------|
| 强效力 | 事实基础坚实、法律依据明确、论证逻辑严密 | 对方自认的事实+直接的对抗法条 |
| 中等效力 | 有一定依据但存在薄弱环节 | 事实有争议或法条适用需解释 |
| 弱效力 | 理论可行但依据不足 | 证据缺失或法条适用存在争议 |

---

## 输出格式要求

1. **标题强制**：报告以 `# 案件抗辩分析报告` 开头，标题前不得有任何内容
2. **三分法分类**：攻防策略推演按权利妨碍→权利消灭→权利限制三类逐类检视
3. **抗辩合并展示**：每项抗辩下直接展示被告抗辩、原告反驳和同类情形检验
4. **抗辩效力标注**：使用"强效力/中等效力/弱效力"文字标注，不得使用符号
5. **空类标注**：某类抗辩不存在时，标注"经审查，本案不存在权利XX抗辩事由"
6. **验证完整**：每项抗辩须通过同类情形检验
7. **内容区分**：串联模式跳过案件概况和请求权基础详情
8. **表格使用**：合同效力审查、抗辩效力汇总使用表格；抗辩策略使用文字格式
9. **禁止代码块包裹**：整个输出禁止用代码块包裹，直接以Markdown格式输出
10. **禁止元信息输出**：输出必须是纯粹的分析内容，禁止附加模块署名、引用标注、生成来源说明
