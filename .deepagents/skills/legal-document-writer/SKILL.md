# 法律文书生成 Skill

根据系统提供的案件字段数据，生成符合中国民事诉讼司法规范的正式法律文书。

## 工作流程

1. **确定文书类型** — 从上下文中识别当前要生成的文书类型，精确匹配下方索引表中的名称
2. **读取写作规范** — 根据索引表找到对应的 reference 文件，**必须读取全文**
3. **应用写作规则** — 严格遵守 reference 中的结构要求、板块写作指南、文书排版模板
4. **填充字段数据** — 将系统提供的字段数据填入文书模板，缺失字段使用 `【待补充：具体字段名】` 占位
5. **质量自检** — 按通用规则和 reference 中的自检清单核验后输出

## 通用写作规则

以下规则适用于所有文书类型，生成任何文书时都必须遵守：

- **输出纯净**：文书必须以一级标题（`# 文书名称`）开头，标题前禁止任何文字、空行、分析过程
- **占位符统一**：缺失的具体数据使用 `【待补充：具体字段名】`，不得留空，不得使用「同上」
- **当事人信息完整**：每个当事人信息独立完整展开，严禁「同上」「同前」
- **文风理性克制**：禁止情绪化语言、道德评判、攻击性措辞，聚焦法律要件
- **禁止杜撰**：材料未提供的信息不得推测，统一用 `【待补充】` 占位
- **排版干净**：通过加粗标签（`**原告：**`）区分板块，禁止使用「一、首部」「二、正文」等结构性小标题出现在文书正文中
- **禁止输出推理过程**：不输出「现在开始生成」「AI分析」「请律师确认」等任何对话性文字
- **全文单一 H1**：文书全文有且仅有一个一级标题 `#`，正文内部使用 `##` 及以下层级

## Reference 文件结构约定

每个 reference 文件统一采用以下结构，确保 Claude 能快速定位所需信息：

1. **文书概述** — 该文书是什么、适用场景
2. **角色设定** — 生成该文书时 AI 应扮演的角色和文风
3. **结构规范** — 文书必须包含的板块及各板块顺序
4. **各板块写作指南** — 每个板块的撰写要点、常见误区、参考示例
5. **文书排版模板** — 输出时的格式化模板（可直接使用或参考）
6. **质量自检清单** — 生成前的核验要点（3-6条）

如果索引中列出的 reference 文件尚不存在，按以下优先级处理：
1. 查找同类别中已有的 reference 文件作为参考
2. 根据通用写作规则 + 文书类型常识尽力生成
3. 在生成结果末尾标注 `[注意：该文书类型的详细写作规范文件尚未创建，本稿基于通用规则生成]`
