# 图表说话

## 核心原则

"用图表说话"是诉讼可视化的核心理念，即通过图表本身的结构、设计和布局，让图表直接传达核心观点和论证逻辑，减少对文字说明的依赖。

## 图表说话的实现方式

### 1. 结构说话

通过图表的整体结构直接体现论证思路：

**示例**：
- 将双方行为分置于时间轴上下两侧 → 自然形成对比
- 用闭环结构展示贸易循环 → 体现融资性贸易特征
- 用层级结构展示股权控制关系 → 体现实际控制人

### 2. 位置说话

通过元素在图表中的位置体现重要性或逻辑关系：

- 中心位置：核心主体或核心争议焦点
- 前置位置：前置条件或基础事实
- 对称位置：相互对立的观点或行为
- 连贯位置：同一性质或同一主体的事项

### 3. 线条说话

通过线条的不同运用表达不同含义：

- 实线：确定的关系或行为
- 虚线：潜在的关系或待证事实
- 粗线：主要关系或重点内容
- 细线：次要关系或辅助内容
- 箭头：方向性（资金流向、权利方向等）

### 4. 颜色说话

通过颜色编码传递信息和情感：

- 蓝色系：客观、中性的信息
- 红色系：违约、争议、需要关注的问题
- 绿色系：合规、正常履行的行为
- 灰色系：背景信息或次要内容
- 对比色：突出对比关系

## 图表说话的设计技巧

### 1. 突出核心观点

**方法**：
- 将核心要素置于视觉中心
- 使用醒目的颜色或线条
- 放大核心元素的尺寸
- 减少干扰性元素

### 2. 简化信息传递

**方法**：
- 删除非必要信息
- 合并同类项
- 使用简洁的标注
- 避免过多的文字说明

### 3. 利用视觉心理学

**方法**：
- 利用人们的阅读习惯（从左到右、从上到下）
- 利用人们对圆形的偏好（代表完整、和谐）
- 利用人们对中心位置的注意力

### 4. 引导视线流动

**方法**：
- 用箭头引导阅读顺序
- 用颜色渐变引导视线
- 用大小变化引导注意力
- 用空白区域分隔不同内容

## 图表说话的常见误区

### 1. 过度依赖文字

图表应该自己说话，而不是变成"配图的文字说明"。如果需要大量文字说明，说明图表本身的设计可能有问题。

### 2. 设计过度复杂

为了说话而过度设计，反而让图表难以理解。好的设计应该"恰到好处"。

### 3. 失去客观性

提交给法官的图表，虽然可以突出己方观点，但不能歪曲客观事实。

### 4. 忽视受众

图表应该考虑受众的阅读习惯和理解能力，而非展示绘图者的技巧。

## 图表说话的实践建议

1. **先画草图**：先不考虑美化，专注于结构和逻辑
2. **反复修改**：多看几次，想象自己第一次看到这张图表时的感受
3. **请他人测试**：请不了解案件的人看图表，看他们能否理解
4. **控制元素数量**：一张图表中的核心元素不宜过多
5. **保持一致性**：同一案件中多张图表的设计风格应保持一致

## 图表说话与文字的配合

虽然强调"图表说话"，但并不意味着完全不需要文字。合理配合的方式：

- **图表说"什么"**：用图表展示事实和关系
- **文字说"为什么"**：用文字说明法律分析和论证
- **标注说"关键点"**：用简短的标注指出图表中的重点

最佳状态是：法官看图表就能理解主要事实和关系，阅读代理意见时能联想到图表，两者相互强化。
