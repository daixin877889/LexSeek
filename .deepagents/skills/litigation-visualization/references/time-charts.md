# 时间图设计方法

## 时间图适用情形

时间图适用于以下案件：
1. 核心要素为时间的案件
2. 事件发生的先后顺序对案件定性有重要影响
3. 需要梳理复杂的程序性事实
4. 需要展示时间区间概念（如保证期间、诉讼时效等）
5. 不存在多个主体间复杂法律关系，且数据非争议焦点

## 时间图基本架构

最简单的时间图架构：在纸的中间横着或竖着画一条线，在上面标注时间和事件即可。

```
时间图（横向示例）：
      ┌─────────────────────────────────────────┐
      │                                         │
      │   事件1   事件2   事件3   事件4         │
时间轴 ────────────────────────────────────────
      │                                         │
      │                                         │
      └─────────────────────────────────────────┘
```

## 时间图的纵向划分

### 一、表达对比关系

利用时间轴上下的天然空间区分，表达不同主体或不同行为的对比。

#### 1. 不同主体行为的对比

将不同主体的行为分置于时间轴上下两侧，自然形成对比效果。

**适用情形**：
- 双方当事人行为对比，突出一方违约
- 多方当事人行为对比，梳理各方责任

**示例**：买卖合同纠纷中，上方展示卖方履行行为，下方展示买方履行行为，红色标注违约行为。

#### 2. 同一主体不同行为的对比

将同一主体的不同性质行为或约定行为与实际履行行为进行对比。

**示例**：房地产合作开发案件中，上方展示按合同约定应付的款项，下方展示实际支付的款项，通过对比展示违约情况。

### 二、行为分层

当案件事实复杂时，将时间轴纵向空间分为更多层次，按照逻辑顺序分布行为。

**分层方式**：
- 按主体分层：不同主体对应不同空间层次
- 按行为性质分层：如申请行为 vs 裁判行为
- 按法院层级分层：基层院、中院、高院的裁判行为

**颜色配合**：给不同的行为类别进行颜色区分，进一步增强层次感。

## 时间图的横向划分

### 一、用括号表达时间区间

括号方式简单有效，常用于表达时间区间概念。

**适用情形**：
- 保证期间、诉讼时效等法律期间
- 合同履行期限
- 法院暂缓受理等特殊期间

**示例**：在时间轴上标注保证期间范围，直观展示债权主张是否在期间内。

### 二、用箭头表达时间区间

箭头可以更直观地表达时间区间的起止和影响。

**适用情形**：
- 需要表达多个时间区间的交叉关系
- 需要表达某段时间的法律后果

**示例**：在时间轴上方用箭头标注法院暂缓受理期间，展示债权到期日恰好落在该期间内。

### 三、用变换线条颜色表达时间区间

通过变换时间轴线不同阶段的颜色，表现不同时间区间内的特定情况。

**适用情形**：
- 某一主体在不同时间区间内使用不同公章
- 某一性质的状态在不同时间段发生变化

**优势**：简洁明了，不会因为箭头或括号使图表混乱。

## 流程图

### 流程图与时间图的区别

- **时间图**：重点是具体时间节点，展示"何时"发生
- **流程图**：重点是先后顺序，展示"如何"进行

### 流程图适用情形

1. 程序性事项的说明（如再审流程、执行异议流程）
2. 多种路径选择的展示（如案外人救济途径）
3. 不需要具体时间节点，只需知道先后顺序的事项

### 流程图绘制要点

1. **给不懂的人看**：流程图是给不熟悉该领域的人看的，要简单易懂
2. **统一入口、区分情形**：给读者一个统一的入口，根据不同情况告诉其选择和后果
3. **避免过于复杂**：不要在一个流程图中展示过多分支和细节

## 时间图设计注意事项

1. **合理利用空间**：根据案件复杂程度，合理划分纵向和横向空间
2. **避免过度拥挤**：事件过多时考虑多张图表或简化处理
3. **颜色编码一致**：同一案件中多张图表的颜色编码应保持一致
4. **突出重点**：用颜色、大小、位置等方式突出关键事件或争议焦点
5. **保持逻辑清晰**：确保图表中的时间顺序和逻辑关系准确无误
