# 判决趋势预测分析方法论

## 核心原则

判决趋势预测以"法官裁判视角"为主线，基于当事人陈述的事实与理由，运用法律逻辑推演预测可能的裁判方向。

分析方法遵循四个基本原则：

1. **法官视角为主线的预判原则**：预测不是从律师视角分析"我方主张是否合理"，而是从法官视角模拟"法官会如何审理这个案子"
2. **仅基于陈述原则**：只审查当事人陈述的事实与理由，不涉及证据评估
3. **要件成就导向原则**：预测结论应基于要件成就状态推导——全部要件成就则支持诉请，有要件不成就则驳回诉请
4. **定性分析原则**：预测必须是定性逻辑分析，禁止量化数据捏造

## 与其他模块配合

本模块职责：基于其他模块的输出结果，进行法律合理性审查、要件成就分析、举证风险评估、判决趋势预测。

**输入来源**：
- 案情概要 → 案件概要模块（本模块直接引用，不重复）
- 时间线 → 大事记模块（本模块不写）
- 争议焦点分析 → 请求权基础模块（本模块引用结果，不重复分析）
- 要件事实清单 → 请求权基础模块（本模块直接使用）

**输出规范**：
- 开头仅简短列明核心请求权基础，不写详细案情
- 不写时间线、大事记
- 直接从法律合理性审查/要件成就分析开始
- 不显式标注"详见某模块"

## 使用流程

本模块在请求权基础分析完成后调用：

1. 接收要件事实清单（来自请求权基础模块）
2. 法律合理性审查 → 审查原被告主张合理性、识别举证风险点
3. 要件成就分析 → 将事实代入要件，判断成就状态
4. 举证责任分配分析 → 罗森贝克法则，评估举证风险
5. 证明标准分析 → 高度盖然性/排除合理怀疑
6. 判决趋势预测 → 基于要件成就推导裁判方向

**条件触发**：
- 涉及数字 → 执行定量核查
- 法律时效存疑 → 执行法律时效审查
- 诉讼时效抗辩 → 执行诉讼时效审查
- 新旧法更替 → 执行新旧法适用分析
