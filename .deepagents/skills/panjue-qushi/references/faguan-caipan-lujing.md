# 法官裁判路径预判法

律师预测判决趋势，核心是模拟法官的裁判思维路径——从生活事实到判决的完整审理流程。

## 核心原则

### 以法官视角为主线的预判原则

预测判决趋势不是从律师视角分析"我方主张是否合理"，而是从法官视角模拟"法官会如何审理这个案子"。律师必须像法官一样思考，才能有效预判判决方向。

### 八步法审理路径

法官裁判不是随意的推理，而是固定的八步流程：

1. **生活事实** → 从当事人陈述中提取原始纠纷事实
2. **法律关系** → 识别纠纷的法律性质（合同/侵权/物权等）
3. **诉讼标的** → 从法律关系派生请求权
4. **案由** → 给诉讼标的贴标签，确定审理范围
5. **核心法条** → 找到请求权基础
6. **要件事实** → 拆分法条为要件
7. **举证责任** → 按罗森贝克法则分配
8. **涵摄裁判** → 将事实代入要件，作出判决

律师应在预测前完成这八步预判，才能准确预测法官的裁判方向。

## 预判步骤

### 第一步：提取生活事实

**动作**：从案情中提取原始纠纷事实，识别关键时间点、主体、行为、争议点。

**需要识别的内容**：
- 争议主体（谁与谁）
- 争议行为（发生了什么）
- 争议时间（何时发生）
- 争议标的（争什么）
- 争议焦点（核心矛盾是什么）

### 第二步：识别法律关系

**动作**：判断纠纷属于何种法律关系类型。

**常见类型判断**：
| 事实特征 | 可能的法律关系 |
|----------|----------------|
| 有合同/协议 | 合同关系 |
| 无合同但有损害 | 侵权关系 |
| 无原因获得利益 | 不当得利 |
| 无委托管理他人事务 | 无因管理 |
| 物的归属争议 | 物权关系 |
| 身份/财产争议 | 婚姻家庭关系 |

### 第三步：确定诉讼标的

**动作**：从法律关系派生具体的请求权类型。

**请求权类型**：
- 给付之诉：请求对方履行义务（支付、交付、行为）
- 确认之诉：请求确认法律关系存在或不存在
- 形成之诉：请求变更或消灭法律关系（撤销、解除）

### 第四步：锁定案由

**动作**：确定案件的具体案由，明确审理范围。

**案由锁定规则**：
- 案由基本等于诉讼标的，基本等于审理范围
- 案由选错可能导致裁定驳回或判决驳回
- 应精确到四级案由（如"买卖合同纠纷"而非"合同纠纷"）

### 第五步：找到核心法条

**动作**：找到请求权基础法条——必须是完整法条（有假定、有结果）。

**核心法条检查**：
- 是否完整法条？（有"如果...则..."结构）
- 是否宣示性条款？（如"应当诚实守信"——无法律后果，不能作为请求权基础）
- 是否有司法解释补充？

### 第六步：拆分要件事实

**动作**：从核心法条拆分要件事实——缺一项，败诉风险增一分。

**要件事实示例**：
| 案由类型 | 典型要件事实 |
|----------|--------------|
| 过错侵权 | 侵权行为、过错、因果关系、损害后果 |
| 合同违约 | 合同成立、合同有效、违约行为、损害后果 |
| 不当得利 | 获得利益、致人损失、无法律根据 |

### 第七步：分配举证责任

**动作**：按罗森贝克法则，对每个要件分配举证责任。

详见 [references/juzheng-fenpei.md]（请求权基础模块）及本模块的举证责任分配分析。

**举证分配规则**：
- 权利发生要件 → 主张权利方举证（通常是原告）
- 权利障碍要件 → 主张障碍方举证（通常是被告）
- 权利消灭要件 → 主张消灭方举证（通常是被告）

### 第八步：涵摄裁判预判

**动作**：模拟法官将认定事实代入要件事实，判断要件是否成就。

详见 [references/hanshe-panduan.md]

**涵摄预判逻辑**：
```
要件事实 + 认定事实 → 成就判断
    │
    ├── 全部要件成就 → 支持诉请
    │
    └── 有要件不成就 → 驳回诉请（部分或全部）
```

## 注意事项

- 八步法是法官审理的固定流程，律师必须按此流程预判
- 每一步出错都可能导致后续预判错误
- 第五步找核心法条时，必须确认是完整法条而非宣示性条款
- 第六步要件拆分要完整，缺项意味着败诉风险
- 第七步举证责任分配直接影响预测结果——关键要件举证困难的一方败诉风险高
