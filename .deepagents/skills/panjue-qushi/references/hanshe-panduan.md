# 涵摄判断操作法

涵摄是法官裁判的核心操作——将认定的法律事实代入要件事实，判断要件是否成就。

## 核心原则

### 目光往返原则

法官裁判是"目光在规范和事实之间往返徘徊"的过程：
- 先看规范（法条要件是什么）
- 再看事实（案件事实是什么）
- 再看规范（事实能否归入要件）
- 再看事实（是否有遗漏事实）
- 最终判断（要件是否成就）

### 涵摄不是简单套法条

涵摄不是"事实符合法条就成立"的简单判断，而是逐步分解、逐项检验的结构化操作：
1. 拆分要件事实（规范层面）
2. 提取认定事实（事实层面）
3. 逐项对应检验（归入判断）
4. 判断成就状态（结论得出）

## 操作步骤

### 第一步：列出要件事实清单

**动作**：从核心法条拆分出的要件事实，形成检验清单。

**示例**：侵权责任（民法典1165条过错责任）
- 要件1：侵权行为（行为人实施了侵害行为）
- 要件2：过错（行为人主观上有过错）
- 要件3：因果关系（侵害行为与损害之间存在因果联系）
- 要件4：损害后果（受害人遭受了实际损害）

### 第二步：提取认定事实

**动作**：基于当事人陈述和证据，提取法官可能认定的法律事实。

**事实提取原则**：
- 仅基于陈述：不涉及证据真实性评估
- 去除冗余：保留与要件相关的事实
- 明确争议点：标注哪些事实有争议

**示例**：
- 事实A：被告驾驶车辆撞伤原告（对应要件1）
- 事实B：被告闯红灯（对应要件2过错）
- 事实C：原告腿部骨折、车辆损坏（对应要件4损害）
- 争议事实D：闯红灯是否是唯一原因（要件3因果关系有争议）

### 第三步：逐项涵摄检验

**动作**：对每个要件进行"事实能否归入要件"的检验。

**涵摄检验格式**：

**要件1（侵权行为）**：
- 要件定义：行为人实施了侵害他人权益的行为
- 认定事实：被告驾驶车辆撞伤原告
- 涵摄判断：驾驶车辆撞伤行人，属于侵害人身权益的行为 → **成就**

**要件2（过错）**：
- 要件定义：行为人主观上有故意或过失
- 认定事实：被告闯红灯
- 涵摄判断：闯红灯违反交通规则，属于过失 → **成就**

**要件3（因果关系）**：
- 要件定义：侵害行为与损害之间存在因果联系
- 认定事实：闯红灯→撞击→骨折（争议：原告是否也有过错）
- 涵摄判断：闯红灯是主要原因，但原告可能也有次要过错 → **部分成就/需进一步认定**

**要件4（损害后果）**：
- 要件定义：受害人遭受了实际损害
- 认定事实：腿部骨折、车辆损坏、医疗费5万元
- 涵摄判断：人身伤害和财产损失，属于实际损害 → **成就**

### 第四步：判断要件成就状态

**动作**：汇总各要件成就状态，得出整体判断。

**成就状态分类**：
| 状态 | 含义 | 对诉请的影响 |
|------|------|--------------|
| 完全成就 | 事实明确归入要件 | 支持该要件 |
| 部分成就 | 部分事实归入，有争议 | 需进一步举证或认定 |
| 不成就 | 事实无法归入要件 | 阻却诉请 |
| 待认定 | 关键事实有争议 | 取决于举证和法官心证 |

**要件成就矩阵**：

| 要件 | 成就状态 | 举证方 | 举证难度 | 风险评估 |
|------|----------|--------|----------|----------|
| 要件1 | 完全成就 | 原告 | 低 | 无风险 |
| 要件2 | 完全成就 | 原告 | 低 | 无风险 |
| 要件3 | 待认定 | 原告/被告 | 中 | 有争议风险 |
| 要件4 | 完全成就 | 原告 | 低 | 无风险 |

### 第五步：推导涵摄结论

**动作**：基于要件成就状态，推导裁判结论。

**结论推导逻辑**：

```
全部要件完全成就 → 支持诉请（或支持主要部分）
    │
    ├── 有要件部分成就 → 部分支持（比例取决于争议程度）
    │
    ├── 有要件不成就 → 驳回诉请（该要件阻却）
    │
    └── 有要件待认定 → 结论取决于举证结果
            │
            ├── 举证方举证成功 → 成就
            └── 举证方举证失败 → 不成就（承担举证不利后果）
```

## 注意事项

- 涵摄判断必须逐项进行，不能跳跃
- "部分成就"和"待认定"状态需要进一步分析举证责任
- 举证责任分配决定"待认定"要件的最终走向——举证困难方承担不利后果
- 涵摄结论要与法律合理性审查和举证责任分析保持一致
- 如果要件成就状态有争议，预测应说明"取决于XX的认定"
