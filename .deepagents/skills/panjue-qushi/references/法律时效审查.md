# 法律时效性审查法

法律知识必须被假定为"可能已过时"，每次分析都需执行以"当前时间"和"法律事实发生时间"为锚点的动态审查流程。

## 审查步骤

### 第一步：锚定法律事实发生日

**动作**：从案情中识别关键法律事实的发生日期（或大致时间段），作为主要时间锚点。

**需要识别的时间点**：
- 合同签订日期
- 履行期限
- 侵权行为发生时间
- 权利主张时间

### 第二步：执行基础框架检索

**动作**：执行常规检索，识别在"法律事实发生日"之前已经生效的、作为基础法律框架的法律和司法解释。

**检索范围**：
- 相关法律条文
- 相关司法解释
- 相关行政法规

### 第三步：执行增量更新强制检索

**动作**：这是最关键的一步。完成基础检索后，立即执行专门的"增量"搜索，检查从"法律事实发生日"到"今天"这段时间内的法律更新。

**检索关键词模板**：
- "（核心案由）最新司法解释"
- "（核心案由）2024年规定"或"2025年规定"
- "（核心法律）修正案"

**必须检查的内容**：
- 新颁布的法律
- 法律修正案
- 新司法解释
- 废止的旧规定

### 第四步：整合与适用分析

**动作**：整合基础框架和增量更新的所有结果。

**适用分析要点**：
- 新司法解释是否有溯及力
- 如何影响旧规则的适用
- 是否需要更新分析框架

## 审查流程图

```
识别法律事实发生日
        │
        ▼
基础框架检索（事实发生日前生效的法律）
        │
        ▼
增量更新检索（事实发生日到今天的更新）
        │
        ▼
整合分析 → 新旧法适用规则 → 司法解释适用规则
        │
        ▼
确定最终适用的法律框架
```

## 输出要求

审查完成后应明确记录：

| 项目 | 内容 |
|------|------|
| 法律事实发生日 | 具体日期或时间段 |
| 基础法律框架 | 事实发生时生效的法律 |
| 增量更新发现 | 新颁布/修改的法律或解释 |
| 最终适用框架 | 分析所依据的法律 |

## 注意事项

- 每次分析都必须执行，不能因"熟知的法律"而跳过
- 增量检索是核心步骤，必须强制执行
- 发现新法律/解释后立即启动新旧法适用分析
