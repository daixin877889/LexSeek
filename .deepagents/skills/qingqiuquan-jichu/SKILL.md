# 请求权基础分析方法论

## 核心原则

请求权基础分析以"谁得向谁根据何种规范为何种请求"为核心问句，构建民事请求权的系统检视框架。

分析方法遵循三个基本原则：

1. **攻防结构原则**：民事规范按诉讼攻防结构分类，区分产生请求权的主要规范、说明要件的辅助规范、排除效果的防御规范
2. **三层递进原则**：请求权检视依次通过"已产生→未消灭→可行使"三道门槛，每层不通过则后续不必检视
3. **举证对应原则**：积极要件由原告举证，消极抗辩由被告举证，举证分配由实体法的要件-抗辩结构决定

## 关键要点

### 一、规范分类
详见 [references/规范分类.md](./references/规范分类.md)

- 主要规范：产生请求权的规范，法律效果为"有权请求..."
- 辅助规范：说明主要规范的构成要件或法律效果
- 防御规范：排除或限制另一规范的法律效果
- 参引规范：具有请求权基础外观但实质指向其他规范

### 二、检视结构
详见 [references/三层四步.md](./references/三层四步.md)

- 第一层：请求权已产生（步骤1积极要件+步骤2未发生抗辩）
- 第二层：请求权未消灭（步骤3消灭抗辩）
- 第三层：请求权可行使（步骤4行使抗辩）

### 三、预选排序
详见 [references/预选排序.md](./references/预选排序.md)

- 标准顺序：合同→类似合同→无因管理→物法→不当得利/侵权
- 排序原则：越特别越在先，构成要件越少越在先

### 四、举证分配
详见 [references/举证责任分配.md](./references/举证分配.md)

- 步骤1积极要件 → 原告举证
- 步骤2/3/4抗辩 → 被告举证
- 反抗辩 → 原告举证

### 五、写作体裁
详见 [references/鉴定体裁.md](./references/鉴定体裁.md)

- 假设：提出"甲得否依第X条向乙请求Y"
- 大前提：列出请求权基础的构成要件
- 涵摄：逐一将事实归入要件
- 结论：回答假设问题

### 六、诉讼框架
详见 [references/法庭报告技术.md](./references/法庭报告技术.md)

- 程序阶段：审查程序要件
- 原告阶段：纯法律审查（假设原告诉述为真）
- 被告阶段：纯法律审查（假设被告诉述为真）
- 证据阶段：仅争议事项举证
- 裁判阶段：形成裁判建议

### 七、竞合处理
详见 [references/竞合处理.md](./references/竞合处理.md)

- 规范排斥竞合：逻辑互斥，适用特别规范
- 请求权规范竞合：一项请求权有多重规范依据

### 八、专门应用
详见 [references/专题应用.md](./references/专题应用.md)

- 不作为侵权：四要件合并判断（详见references/不作为侵权检视.md）
- 合同效力：有效推定原则（详见references/有效推定.md）
- 法律漏洞：类推填补（详见references/漏洞类推.md）
- 体系构建：公因式展开（详见references/公因式展开.md）

## 工具调用规范

### 必须调用的工具

| 工具名称 | 用途 | 调用时机 |
|----------|------|----------|
| search_law | 检索法律规范条文 | 规范检索阶段，在案情分析完成后调用 |

### 工具调用流程

```
案情分析 → [调用 search_law] → 规范分类 → 预选排序 → 三层四步检视 → ...
```

### search_law 调用规范

**调用时机**：在"使用流程"第2步"规范检索"时必须调用

**输入参数**：
- 争议焦点：从案情中提取的核心争议点
- 案情特征标签：如"合同违约""物权侵害""不当得利"等

**输出处理**：
1. 对返回的法条进行规范分类（主要/辅助/防御/参引）
2. 排除明显不成立的候选规范
3. 按预选排序原则排列检视顺序
4. 将分类后的规范用于三层四步检视

**调用示例**：
```
调用：search_law(争议焦点="甲能否请求乙返还占有物", 案情特征=["物权侵害", "占有保护"])
返回：相关法条列表
处理：识别第235条为主要规范，第245条为防御规范...
```

## 使用流程

1. 案情分析 → 提取争议焦点和案情特征标签
2. 规范检索 → **调用 search_law 检索法条**，识别请求权基础规范
3. 规范分类 → 判断主要/辅助/防御规范
4. 预选排序 → 确定检视顺序
5. 三层四步检视 → 系统检视每项请求权
6. 举证分配 → 确定举证主体
7. 竞合处理 → 处理多项规范同时适用
8. 鉴定体裁输出 → 规范写作格式
