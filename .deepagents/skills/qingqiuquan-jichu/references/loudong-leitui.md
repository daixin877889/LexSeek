# 法律漏洞类推填补

遍检所有请求权基础后无适用规范时，以类推填补法律漏洞。

## 定位漏洞

请求权基础思维有双重功能：
- 分析工具：系统检视请求权
- 定位工具：遍检后无果即精确定位法律漏洞

## 基本方法

**类推填补**：在法律评价上相似的案件之间类比，将已适用的规范类推到未规定的案件。

### 类推的"路标功能"

在相似判例中已适用的请求权基础，可指引待决案件的规范找寻方向——不必从零开始找法。

## 执行步骤

1. 确认法律漏洞
   - 遍检所有可能的请求权基础后确认无适用的规范
   - 若找到可适用的规范 → 直接适用，无需类推，终止

2. 寻找类似案件/规范
   - 在法律评价上相似的已规定案件中找出可参照的规范
   - 判断"相似性"的标准：法律评价上的相同点多于不同点

3. 论证类推合理性
   - 说明待决案件与参照案件在法律评价上的核心相同点
   - 说明不同点不构成类推障碍

4. 适用类推结果
   - 将参照规范类推适用于待决案件，得出结论

## 与其他方法的区别

| 方法 | 说明 |
|------|------|
| 类推 | 不同案件间类比（法官的法律续造） |
| 目的性扩张 | 同一条文内扩张解释 |
| 参照适用 | 立法明文规定的"参照适用"（立法授权） |

## 不适用情形

- 有明确法条可适用：直接适用，不需要类推
- 立法政策讨论：讨论"应否立法"而非"如何适用现有法"
- 违反立法目的的类推：类推结果与相关领域的立法目的相悖，不应类推

## 注意事项

- 类推不是"找个差不多的法条凑合用"，必须论证法律评价上的实质相似性
- "法律评价上的相似性"判断标准可能模糊，需要具体领域的专业知识
