# 情形判断法

根据用户提供的材料状况判断分析模式。

## 一、情形分类

根据用户提供的材料，分为四种情形：

| 情形 | 判断标准 | 分析模式 |
|------|----------|----------|
| 情形一 | 未提供具体证据文件 | 推演模式 |
| 情形二 | 提供少量、零散证据 | 混合模式 |
| 情形三 | 提供较充分的真实证据 | 实证模式 |
| 情形四 | 提供法院判决文书 | 判决分析模式 |

## 二、情形判断标准

### 情形一：推演模式

**判断标准**：
- 用户仅提供案情描述
- 未上传任何证据文件
- 或仅提及证据名称但无具体内容

**分析任务**：
1. 根据案情推演所需证据类型
2. 构建证据清单框架
3. 模拟证据链与策略分析
4. 制定取证计划

**输出重点**：
- 建议证据清单
- 各证据的预期证明力
- 取证关键标准
- 模拟证据链构建

### 情形二：混合模式

**判断标准**：
- 用户提供了部分证据文件
- 证据数量较少（通常少于5份）
- 证据可能零散、不完整

**分析任务**：
1. 审查已有证据
2. 识别证据缺口
3. 整合已有证据与待补证据
4. 制定补强策略

**输出重点**：
- 现有证据的三性审查
- 现有证据的证明力评估
- 建议补充证据清单
- 混合拼图式证据链

### 情形三：实证模式

**判断标准**：
- 用户提供了较充分的证据文件
- 证据数量较多（通常5份以上）
- 证据覆盖主要待证事实

**分析任务**：
1. 全面审查各项证据
2. 构建完整证据链
3. 评估证据链强度
4. 制定诉讼策略

**输出重点**：
- 真实证据清单
- 详细的三性审查
- 完整的证据链分析
- 实战策略建议

### 情形四：判决分析模式

**判断标准**：
- 用户提供了法院判决书
- 需要分析法院的证据认定

**分析任务**：
1. 提取判决中的证据认定
2. 分析法院的证据评价
3. 总结裁判规则

**输出重点**：
- 证据认定分析表
- 法院认证理由
- 裁判规则提炼

## 四、情形判断注意事项

### 内部判断原则

**重要**：情形判断是内部工作逻辑，不应在输出中暴露：
- 不输出"模式判断""场景A/B"等表述
- 不输出"评估为稀疏证据/充分证据"的过程
- 不输出内部判断路径

### 统一输出外观

虽然内部判断不同，但对外输出应统一：
- 统一从"# 证据清单梳理"开始
- 统一输出证据清单+证据链分析+策略建议
- 统一以"重要提示"结尾

### 判断边界

**证据数量判断**：
- 少于5份：视为零散
- 5-10份：需综合评估
- 10份以上：视为较充分

**证据覆盖度判断**：
- 是否覆盖主要待证事实
- 是否形成基本证据链
- 是否存在明显缺口

## 五、各情形的处理差异

| 处理环节 | 情形一 | 情形二 | 情形三 | 情形四 |
|----------|--------|--------|--------|--------|
| 证据清单 | 全部建议 | 已有+建议 | 全部已有 | 法院认定 |
| 三性审查 | 不适用 | 已有证据审查 | 全面审查 | 法院评价 |
| 证明力 | 预期评估 | 已有评估+预期 | 实际评估 | 法院认定 |
| 证据链 | 模拟构建 | 混合构建 | 实际构建 | 认定分析 |
| 策略 | 取证为主 | 补强为主 | 实战为主 | 规则总结 |


## 七、常见失败模式

| 失败模式 | 说明 | 正确做法 |
|----------|------|----------|
| 情形判断外泄 | 在输出中暴露"推演模式"等内部判断 | 统一输出外观，不暴露内部逻辑 |
| 情形判断错误 | 将少量证据判断为充分证据 | 按数量标准判断（<5份为零散） |
| 输出格式不一致 | 不同情形使用不同输出格式 | 统一从"# 证据清单梳理"开始 |
| 情形一过于详细 | 无证据时仍详细分析三性 | 情形一不适用三性审查，推演证据类型 |
| 情形四遗漏法院认定 | 未提取判决中的证据认定 | 逐项提取法院认证理由 |

## 八、实务操作要点

### 快速判断方法

1. **看材料类型**：是否为判决书？→ 情形四
2. **看证据数量**：是否少于5份？→ 情形一或二
3. **看证据覆盖**：是否覆盖主要待证事实？→ 判断充分性

### 判断后的分析路径

| 情形 | 分析起点 | 核心任务 |
|------|----------|----------|
| 情形一 | 待证事实分解 | 推演证据类型 |
| 情形二 | 已有证据审查 | 识别缺口+补强建议 |
| 情形三 | 全面证据审查 | 证据链构建+策略制定 |
| 情形四 | 判决书提取 | 认定分析+规则总结 |
