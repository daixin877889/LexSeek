# 设计文档：法律条文层级排序算法

## 概述

本功能为法律条文管理系统提供正确的层级排序算法。算法采用树形递归排序方法，支持复杂的层级结构，包括非层级类型（notice、header、footer、annex）和层级类型（L1-L5），并能正确处理跳级情况（如章直接包含条，没有节）。

排序算法的核心思想是：
1. 按父级路径分组条文
2. 在每个组内按 order 字段排序
3. 使用深度优先遍历构建最终排序列表

## 架构

### 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    法律条文排序系统                          │
│                                                               │
│  ┌───────────────────────────────────────────────────────┐  │
│  │              输入：未排序的条文列表                    │  │
│  └───────────────────────────────────────────────────────┘  │
│                            │                                  │
│                            ▼                                  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │         步骤 1：计算父级路径和节点路径                 │  │
│  │         - getParentPath(article)                       │  │
│  │         - getNodePath(article)                         │  │
│  └───────────────────────────────────────────────────────┘  │
│                            │                                  │
│                            ▼                                  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │         步骤 2：按父级路径分组                         │  │
│  │         - Map<parentPath, articles[]>                  │  │
│  └───────────────────────────────────────────────────────┘  │
│                            │                                  │
│                            ▼                                  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │         步骤 3：组内按 order 排序                      │  │
│  │         - group.sort((a, b) => a.order - b.order)      │  │
│  └───────────────────────────────────────────────────────┘  │
│                            │                                  │
│                            ▼                                  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │         步骤 4：深度优先遍历                           │  │
│  │         - visit(parentPath)                            │  │
│  │         - 递归访问子节点                               │  │
│  └───────────────────────────────────────────────────────┘  │
│                            │                                  │
│                            ▼                                  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │              输出：排序后的条文列表                    │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 数据流

```
条文列表
    │
    ├─> 非层级类型（notice, header, footer, annex）
    │   └─> 父级路径 = ''
    │       └─> 节点路径 = `__${type}__${id}`
    │
    └─> 层级类型（l1, l2, l3, l4, l5）
        ├─> l1: 父级路径 = '', 节点路径 = l1
        ├─> l2: 父级路径 = l1, 节点路径 = l1/l2
        ├─> l3: 父级路径 = l1/l2, 节点路径 = l1/l2/l3
        ├─> l4: 父级路径 = l1/l2/l3, 节点路径 = l1/l2/l3/l4
        └─> l5: 父级路径 = l1/l2/l3/l4, 节点路径 = l1/l2/l3/l4/l5
```

## 组件和接口

### 核心函数

#### 1. sortArticlesByHierarchy

**职责**：主排序函数，接收条文列表，返回排序后的列表。

**签名**：
```typescript
function sortArticlesByHierarchy(articles: Article[]): Article[]
```

**算法流程**：
1. 遍历所有条文，计算父级路径
2. 按父级路径分组
3. 在每个组内按 order 排序
4. 从根节点（空路径）开始深度优先遍历
5. 返回排序后的列表

**时间复杂度**：O(n log n)，其中 n 是条文数量
**空间复杂度**：O(n)

#### 2. getParentPath

**职责**：计算条文的父级路径。

**签名**：
```typescript
function getParentPath(article: Article): string
```

**规则**：
- 非层级类型：返回 `''`
- l1：返回 `''`
- l2：返回 `l1`
- l3：返回 `l1/l2`（如果 l2 存在）或 `l1`（如果 l2 不存在，跳级）
- l4：返回 `l1/l2/l3`
- l5：返回 `l1/l2/l3/l4`（如果 l4 存在）或 `l1/l2/l3`（如果 l4 不存在，跳级）

**示例**：
```typescript
// 正常层级
getParentPath({ type: 'l3', l1: '第一编', l2: '第一分编', l3: '第一章' })
// 返回：'第一编/第一分编'

// 跳级情况
getParentPath({ type: 'l3', l1: '第一编', l2: null, l3: '第一章' })
// 返回：'第一编'
```

#### 3. getNodePath

**职责**：生成条文的节点路径（唯一标识）。

**签名**：
```typescript
function getNodePath(article: Article): string
```

**规则**：
- 非层级类型：返回 `__${type}__${id}`
- l1：返回 `l1`
- l2：返回 `l1/l2`
- l3：返回 `l1/l2/l3`（跳过 null 字段）
- l4：返回 `l1/l2/l3/l4`（跳过 null 字段）
- l5：返回 `l1/l2/l3/l4/l5`（跳过 null 字段）

**示例**：
```typescript
// 正常层级
getNodePath({ type: 'l3', l1: '第一编', l2: '第一分编', l3: '第一章' })
// 返回：'第一编/第一分编/第一章'

// 跳级情况
getNodePath({ type: 'l3', l1: '第一编', l2: null, l3: '第一章' })
// 返回：'第一编/第一章'

// 非层级类型
getNodePath({ type: 'notice', id: 'abc123' })
// 返回：'__notice__abc123'
```

#### 4. visit

**职责**：深度优先遍历辅助函数。

**签名**：
```typescript
function visit(parentPath: string, grouped: Map<string, Article[]>, result: Article[]): void
```

**算法**：
1. 获取指定父级路径下的所有子节点
2. 遍历每个子节点：
   a. 将子节点添加到结果列表
   b. 计算子节点的节点路径
   c. 递归调用 visit，访问子节点的子节点

### 数据模型

#### Article 类型

```typescript
interface Article {
  id: string
  type: ArticleType
  l1?: string | null
  l2?: string | null
  l3?: string | null
  l4?: string | null
  l5?: string | null
  order: number | null
  content?: string | null
}

type ArticleType = 'notice' | 'header' | 'footer' | 'annex' | 'l1' | 'l2' | 'l3' | 'l4' | 'l5'
```

## 正确性属性

*属性是一个特征或行为，应该在系统的所有有效执行中保持为真——本质上是关于系统应该做什么的形式化陈述。属性作为人类可读规范和机器可验证正确性保证之间的桥梁。*

### 属性 1：非层级类型顶层处理

*对于任何*非层级类型的条文（notice、header、footer、annex），其父级路径应为空字符串，节点路径应为 `__${type}__${id}` 格式，且在排序后应按 order 字段升序排列。

**验证：需求 1.1, 1.2, 1.3, 1.4**

### 属性 2：父级路径计算正确性

*对于任何*条文，其父级路径应根据类型和层级字段正确计算，且跳过所有 null 字段。

**验证：需求 4.1, 4.2, 4.3, 4.4, 4.5, 4.6, 4.7**

### 属性 3：节点路径生成正确性

*对于任何*条文，其节点路径应根据类型和层级字段正确生成，且跳过所有 null 字段，非层级类型应使用 `__${type}__${id}` 格式。

**验证：需求 5.1, 5.2, 5.3, 5.4, 5.5, 5.6, 5.7**

### 属性 4：父子关系判断正确性

*对于任何*两个条文，如果一个条文的父级路径等于另一个条文的节点路径，则它们应被判定为父子关系。

**验证：需求 6.1, 6.2, 6.3**

### 属性 5：跳级关系识别

*对于任何*包含跳级的条文结构（l1→l3 或 l3→l5），系统应正确识别跳级关系，且跳级子节点的父级路径应跳过中间层级。

**验证：需求 7.1, 7.2, 7.3, 7.4**

### 属性 6：同层级排序正确性

*对于任何*属于同一父级的条文列表，排序后应按 order 字段升序排列，null 值应视为 0。

**验证：需求 8.1, 8.2, 8.3**

### 属性 7：深度优先遍历顺序

*对于任何*层级树，深度优先遍历应先访问父节点，再按 order 顺序递归访问所有子节点。

**验证：需求 9.1, 9.2, 9.3, 9.4**

### 属性 8：排序结果完整性

*对于任何*条文列表，排序后的结果应包含所有输入条文，且没有重复，每个条文只出现一次。

**验证：需求 10.4, 10.5**

### 属性 9：排序结果层级正确性

*对于任何*排序后的条文列表，所有顶层节点应在前，每个节点的子节点应紧跟其后，且同层级节点应按 order 排序。

**验证：需求 10.1, 10.2, 10.3**

### 属性 10：性能要求

*对于任何*包含 n 个条文的列表，排序算法应在 O(n log n) 时间内完成，且对于 1000 条以内的条文应在 100ms 内完成。

**验证：需求 11.1, 11.2, 11.3**

### 属性 11：空输入处理

*对于任何*空数组输入，排序函数应返回空数组，不抛出异常。

**验证：需求 12.1**

### 属性 12：异常输入处理

*对于任何*包含无效类型或异常数据的输入，排序函数应记录警告并优雅处理，不抛出未捕获的异常。

**验证：需求 12.2, 12.3, 12.4**

## 错误处理

### 1. 空输入

**场景**：输入的条文列表为空数组。

**处理**：
- 直接返回空数组
- 不执行任何计算
- 不记录日志

### 2. 无效类型

**场景**：条文的 type 字段不是有效的 ArticleType。

**处理**：
- 记录警告日志
- 跳过该条文
- 继续处理其他条文

### 3. 缺失必要字段

**场景**：条文缺少 id 或 type 字段。

**处理**：
- 记录错误日志
- 跳过该条文
- 继续处理其他条文

### 4. 循环引用

**场景**：条文的层级关系形成循环（理论上不应该发生）。

**处理**：
- 检测循环引用
- 抛出错误并记录详细信息
- 返回原始顺序

### 5. 性能超时

**场景**：排序时间超过预期（可能是数据量过大或算法问题）。

**处理**：
- 记录警告日志
- 继续执行（不中断）
- 返回结果

## 实现注意事项

### 1. 路径分隔符

使用 `/` 作为路径分隔符，确保与文件系统路径不冲突。

### 2. Null 值处理

在构建路径时，跳过所有 null 或 undefined 的字段，避免路径中出现 `null` 字符串。

### 3. Order 字段默认值

将 null 的 order 字段视为 0，确保排序的一致性。

### 4. 性能优化

- 使用 Map 而不是对象存储分组，提高查找性能
- 避免重复计算路径，缓存计算结果
- 使用原生 sort 方法，利用 V8 引擎的优化

### 5. 类型安全

- 使用 TypeScript 严格模式
- 为所有函数添加类型注解
- 使用类型守卫验证输入

### 6. 日志记录

- 记录排序开始和结束时间
- 记录条文数量和分组数量
- 记录异常情况和警告

### 7. 向后兼容

- 保持与现有代码的兼容性
- 不修改数据库结构
- 不影响其他功能

## 算法示例

### 示例 1：正常层级结构

**输入**：
```typescript
[
  { id: '1', type: 'notice', order: 1 },
  { id: '2', type: 'l1', l1: '第一编', order: 2 },
  { id: '3', type: 'l2', l1: '第一编', l2: '第一分编', order: 1 },
  { id: '4', type: 'l3', l1: '第一编', l2: '第一分编', l3: '第一章', order: 1 },
  { id: '5', type: 'l3', l1: '第一编', l2: '第一分编', l3: '第二章', order: 2 },
]
```

**处理过程**：
1. 分组：
   - `''`: [notice, l1]
   - `'第一编'`: [l2]
   - `'第一编/第一分编'`: [l3(第一章), l3(第二章)]

2. 组内排序：
   - `''`: [notice(1), l1(2)]
   - `'第一编'`: [l2(1)]
   - `'第一编/第一分编'`: [l3(1), l3(2)]

3. 深度优先遍历：
   - 访问 `''` → notice, l1
   - 访问 `'第一编'` → l2
   - 访问 `'第一编/第一分编'` → l3(第一章), l3(第二章)

**输出**：
```typescript
[
  { id: '1', type: 'notice', order: 1 },
  { id: '2', type: 'l1', l1: '第一编', order: 2 },
  { id: '3', type: 'l2', l1: '第一编', l2: '第一分编', order: 1 },
  { id: '4', type: 'l3', l1: '第一编', l2: '第一分编', l3: '第一章', order: 1 },
  { id: '5', type: 'l3', l1: '第一编', l2: '第一分编', l3: '第二章', order: 2 },
]
```

1. 分组：

2. 深度优先遍历：

### 示例 3：混合类型

**输入**：
```typescript
[
  { id: '1', type: 'notice', order: 1 },
  { id: '2', type: 'header', order: 2 },
  { id: '3', type: 'l3', l1: null, l2: null, l3: '第一章', order: 1 },

```

**处理过程**：
1. 分组：
   - `''`: [notice, header, l3, footer]
   - `'第一章'`: [l5]

2. 组内排序：
   - `''`: [notice(1), header(2), l3(1), footer(3)]

3. 深度优先遍历：
   - 访问 `''` → notice, header, l3, footer
   - 访问 `'第一章'` → l5

**输出**：
```typescript
[
  { id: '1', type: 'notice', order: 1 },
  { id: '2', type: 'header', order: 2 },
  { id: '3', type: 'l3', l1: null, l2: null, l3: '第一章', order: 1 },
```
