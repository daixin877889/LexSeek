# 需求文档

## 简介

为路由权限管理页面（/admin/permissions/routes）增加编辑功能，包括：是否菜单切换、删除操作、排序调整。参考 API 权限管理页面（/admin/permissions/api）的交互模式实现。

## 术语表

- **路由权限页面**: 路由权限管理页面，用于管理前端路由权限资源
- **路由记录**: 包含路由名称、标题、路径、图标、是否菜单、排序等信息的数据实体
- **是否菜单**: 是否作为菜单显示的布尔标识
- **排序值**: 路由的排序值，数值越小越靠前

## 需求列表

### 需求 1: 是否菜单切换

**用户故事:** 作为管理员，我希望能够切换路由的菜单状态，以便控制哪些路由显示在导航菜单中。

#### 验收标准

1. 当管理员查看路由列表时，路由权限页面应在"菜单"列为每个路由显示一个 Switch 开关组件
2. 当管理员切换 Switch 组件时，路由权限页面应发送 API 请求更新 isMenu 状态
3. 当 API 请求成功时，路由权限页面应更新本地状态并显示成功提示消息
4. 如果 API 请求失败，路由权限页面应显示错误提示消息并恢复 Switch 状态

### 需求 2: 删除路由权限

**用户故事:** 作为管理员，我希望能够删除路由权限，以便从系统中移除过时或不需要的路由。

#### 验收标准

1. 当管理员查看路由列表时，路由权限页面应在"操作"列为每个路由显示删除按钮
2. 当管理员点击删除按钮时，路由权限页面应显示包含路由路径的确认对话框
3. 当管理员确认删除时，路由权限页面应发送 DELETE API 请求
4. 当删除成功时，路由权限页面应刷新路由列表并显示成功提示消息
5. 如果删除失败，路由权限页面应显示错误提示消息

### 需求 3: 排序调整

**用户故事:** 作为管理员，我希望能够调整路由的排序顺序，以便控制导航菜单中的显示顺序。

#### 验收标准

1. 当管理员查看路由列表时，路由权限页面应在"排序"列显示可编辑的排序输入框或内联编辑控件
2. 当管理员修改排序值时，路由权限页面应发送 API 请求更新排序值
3. 当 API 请求成功时，路由权限页面应更新本地状态并显示成功提示消息
4. 如果 API 请求失败，路由权限页面应显示错误提示消息并恢复排序值
5. 路由权限页面应验证排序值为非负整数

### 需求 4: 操作列布局

**用户故事:** 作为管理员，我希望有一个统一的操作列布局，以便方便地访问所有路由管理操作。

#### 验收标准

1. 路由权限页面应在路由列表表格中添加"操作"列
2. 路由权限页面应在"操作"列中显示删除按钮
3. 路由权限页面应保持与 API 权限页面一致的样式风格
