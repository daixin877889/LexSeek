## 一、产品概述

### 1.1 产品愿景

**一句话定位：** 专为执业律师打造的AI 办案智能体工作台。

升级文案: 法索 AI拒绝套壳对话，深度贴合诉讼研判逻辑。提供从多模态案情梳理到“双层记忆”长效跟进的专家级深度分析，助您高效决策，掌控全局。

**长期愿景：** 通过"分析引擎 + 智能生成系统 + 协同管理平台"三大支柱，用一个平台彻底解决法律行业的效率痛点，将法律工作化繁为简。

### 1.2 产品定位

LexSeek 是一款面向法律行业的 B2B2C SaaS 平台，服务对象涵盖独立律师、中小律所和企业法务团队。

**核心价值主张：**

| 行业痛点 | 数据佐证 | LexSeek 解决方案 |
|---------|---------|----------------|
| 时间浪费 | 法律专业人士高达 **50%** 的工作时间被耗费在重复性的文书审阅和信息检索上 | **AI 分析引擎**：多模态驱动，自动梳理案情、生成概要、提取关键信息 |
| 经验鸿沟 | **75%** 的青年律师成长和办案质量高度受限于个人经验，行业知识难以标准化传承 | **智能策略预判**：7 大分析模块覆盖从案由确认到判决趋势预测的完整分析链 |
| 人才困局 | 一半的小型律所难以吸引和留住人才，高强度的重复性工作是职业倦怠的主要原因 | **标准化分析流程**：将资深律师的分析思路沉淀为 AI 工作流，降低执业门槛 |

**竞争差异点：**

1. **一站式七模块输出**：单次分析即可生成案情概要、大事记、请求权、趋势预测、案由、抗辩策略、证据清单七项结果，无需在多个工具间切换
2. **全流程材料处理**：支持文本、PDF 文档、图片、音频四种材料格式，AI 自动完成 OCR 识别、语音转写和结构化提取
3. **灵活付费模式**：会员订阅制 + 积分消耗制组合，用户可按需选择，避免一次性高额投入
4. **实时流式交互**：分析过程实时可见（逐字输出），支持中断恢复和深度追问，而非黑箱式等待结果

### 1.3 目标用户画像

#### 画像 A：青年律师（执业 1-5 年）

- **背景**：刚取得执业资格或执业初期，独立办案经验有限
- **痛点**：面对复杂案件不知从何入手；缺乏资深律师的指导和经验传承；花大量时间做基础的案情梳理和法律检索
- **需求**：希望有一个"虚拟资深律师"帮助梳理案情、分析策略、预测风险
- **使用场景**：接到新案件后，上传材料让 AI 快速生成分析报告，作为自己分析的起点和参考

#### 画像 B：中小律所主任

- **背景**：管理 5-20 人的律所，既是管理者也是业务骨干
- **痛点**：人力有限，无法为每个案件投入足够分析时间；新人培养周期长，人才流失率高；需要提升全所的办案效率和质量标准
- **需求**：一个能让全所律师使用的标准化分析工具，降低人均案件成本
- **使用场景**：为律所购买团队版，要求所有律师在办案时使用 AI 分析辅助，提升整体效率

#### 画像 C：企业法务负责人

- **背景**：大中型企业的法务部门负责人，管理大量合同纠纷和劳动争议
- **痛点**：案件量大且类型重复，但每个案件仍需逐一分析；需要快速评估案件风险和胜诉概率；上级决策需要数据支撑
- **需求**：批量化、标准化的案件分析流程，生成可呈报的分析报告
- **使用场景**：将积压的劳动争议案件批量上传分析，快速获得赔偿金额估算和胜诉概率评估

#### 痛点与功能映射

| 用户痛点 | 对应功能模块 | 解决方式 |
|---------|-----------|---------|
| 案情梳理耗时费力 | AI 案情概要生成 | 上传材料后 5 分钟内生成结构化概要 |
| 案件时间线混乱 | 案件大事记提取 | 自动按时间顺序整理关键事件 |
| 法律检索效率低 | 法律知识库（全文+语义搜索） | 支持自然语言查询，AI 理解意图匹配法条 |
| 诉讼策略依赖经验 | 请求权分析、判决趋势预测 | 基于类案数据量化胜诉可能性 |
| 对方抗辩难以预判 | 抗辩分析及应对策略 | AI 预测对方抗辩策略并给出应对建议 |
| 证据整理无从下手 | 证据清单预梳理 | 自动梳理所需证据清单和收集方向 |
| 费用计算易出错 | 10+ 法律计算工具 | 内置权威计算标准，自动分段、自动适用最新利率 |
| 案由选择不准确 | 预选案由 | AI 推荐适用的案由分类 |

## 三、核心功能详述

### 3.1 用户认证体系

#### 3.1.1 用户注册

LexSeek 支持两种注册方式，默认推荐微信小程序扫码：

**方式一：微信小程序扫码注册（推荐）**

用户打开注册页面后，默认显示小程序二维码。使用微信扫码后在小程序内完成注册流程，信息自动同步到网页端。此方式最为便捷，一步完成注册和微信绑定。

**方式二：网站表单注册**

适用于不方便使用微信的场景。流程如下：

1. 用户输入姓名和手机号码
2. 点击"获取验证码"，系统发送短信验证码到该手机
3. 用户输入收到的验证码
4. 设置登录密码
5. 若通过邀请链接访问，邀请码自动填入
6. 点击"注册"完成

**注册福利：** 新用户注册即可获得 **7 天全功能免费试用**（旗舰版权限）以及初始积分赠送，降低体验门槛。

#### 3.1.2 用户登录

主要登录方式为手机号 + 密码登录。页面提供"记住我"选项，勾选后下次访问无需重新输入。登录成功后自动跳转至 Dashboard 首页。

#### 3.1.3 密码管理

- **忘记密码**：在登录页面点击"忘记密码"链接，通过手机号 + 短信验证码验证身份后设置新密码
- **修改密码**：在个人设置的安全页面，需先验证当前密码，再设置新密码

### 3.2 案件管理

#### 3.2.1 AI 辅助创建案件（推荐路径）

这是 LexSeek 最推荐的案件创建方式。用户进入创建页面后，看到欢迎语"你好，我是小索，你的案件分析助手"，以及一个多功能输入区域。

**用户可以进行三种操作：**

1. **自由输入案情描述**：在文本框中用自然语言描述案件情况，例如"张三与李四因房屋租赁合同产生纠纷，张三欠付三个月租金共计 15000 元..."
2. **上传案件材料**：支持上传 PDF 文档、图片、音频等文件，AI 会自动识别和提取内容
3. **选择示例案件**：页面下方展示多个预置的示例案件卡片，用户点击即可一键填充，快速体验完整的分析流程。如果输入框已有内容，选择示例时会弹出替换确认提示，防止误操作

**AI 信息提取流程：**

用户点击"提取信息"后，系统进入 AI 提取阶段：

1. 页面显示全屏加载动画，提示"正在提取案件信息，请稍候..."
2. AI 模型综合分析用户输入的文字和上传的材料内容
3. 自动提取以下结构化信息：
   - **固定字段**：案件标题、原告列表、被告列表、案件类型、案情概要
   - **扩展字段**：根据案件类型动态提取额外信息（如争议标的、涉及地域、合同类型、伤残等级等）
4. 若用户仅上传了文件但未输入文字，系统会自动补充提示文本要求 AI 从文件中提取信息

**提取结果确认：**

AI 提取成功后，自动进入确认编辑表单，AI 提取的信息自动填入各字段：

- 用户可自由修改任何字段（案件类型、当事人信息、案情描述等）
- 可增减原告或被告（支持多方当事人）
- 可关联或删除已上传的材料文件
- 若对提取结果不满意，可点击"返回"回到输入界面重新提取

**提取失败处理：** 若 AI 无法从输入内容中识别到足够的案件信息，系统提示用户补充更多描述或切换到手动创建模式。

**创建完成后的流转：** 用户确认信息后点击"创建案件"，系统立即创建案件记录并**自动跳转到初始化分析页面**，无需用户手动操作，直接进入分析模块选择阶段。

#### 3.2.2 手动创建案件

适用于用户已有完整案件信息、不需要 AI 提取的场景。

点击"手动创建"切换到表单模式，包含以下字段：

| 字段 | 是否必填 | 说明 |
|------|---------|------|
| 案件类型 | 必填 | 从系统预设的案件类型下拉列表中选择 |
| 案件标题 | 选填 | 简要的案件名称 |
| 原告信息 | 至少填一方 | 支持输入多个原告 |
| 被告信息 | 至少填一方 | 支持输入多个被告 |
| 案情内容 | 选填 | 自由文本描述 |
| 关联材料 | 选填 | 选择已上传的文件 |

#### 3.2.3 案件列表

案件列表是用户管理所有案件的主视图，提供两种展示模式：

- **列表视图**（默认）：表格形式展示案件标题、类型、状态、创建时间等字段，适合大量案件的快速浏览
- **卡片网格视图**：以卡片形式排列，每张卡片展示案件核心信息和分析进度，视觉效果更直观

**页面顶部展示统计卡片：** 累计案件数、进行中案件数、已完成案件数。支持按案件状态（进行中/已完成/已关闭）筛选，以及关键词搜索。移动端自动适配为单列卡片流视图。

案件删除操作需要二次确认弹窗，防止误操作。

#### 3.2.4 案件详情（四栏响应式布局）

案件详情页采用精心设计的多栏布局，将案件的所有信息在一个页面中呈现：

**桌面端（三栏布局）：**

```
┌────────────────────────────────────────────────┐
│ 页面头部：返回按钮 │ 案件标题 │ Tab 导航 │ 操作  │
├──────────┬──────────────────┬──────────────────┤
│ 左侧     │                  │ 右侧面板         │
│ 导航菜单 │ 中央内容区域      │ （可调整宽度）   │
│ （可折叠）│ （根据 Tab 切换） │                  │
│          │                  │ - 分析结果卡片   │
│ - 概览   │                  │ - 历史版本       │
│ - 材料   │                  │ - 模块对话       │
│ - 分析   │                  │                  │
│ - 文书   │                  │                  │
└──────────┴──────────────────┴──────────────────┘
```

**各区域功能说明：**

**概览区（Overview）：** 展示案件基本信息（标题、类型、状态）、当事人列表（原告/被告）、创建时间和最后更新时间。提供快速操作按钮：添加材料、开始分析、下载报告。

**材料管理区（Materials）：** 以表格或卡片形式展示所有已上传的案件材料。每份材料显示名称、类型、大小、上传时间和识别状态。支持四种类型的在线预览：

| 材料类型 | 预览方式 | 支持格式 |
|---------|---------|---------|
| 文本 | 文本弹框直接展示 | 纯文本 |
| 文档 | PDF/Office 在线预览器 | PDF、DOC、DOCX |
| 图片 | 图片查看器（支持缩放旋转） | JPG、PNG、GIF |
| 音频 | 音频播放器（显示波形和时长） | MP3、WAV、M4A |

材料上传后系统自动启动识别处理，状态实时更新：待处理 → 处理中 → 已完成/失败。

**分析结果区（Analysis）：** 提供两种查看模式——**仪表盘模式**以卡片网格展示 7 个模块的状态和摘要，**详情模式**提供沉浸式阅读界面。每个模块支持"重新生成""重试"（针对失败模块）和"模块对话"（深度追问）操作。

**模块对话区：** 在分析结果下方提供输入框，用户可以对任意模块的分析结果发起追问。例如在查看抗辩分析结果后，可以继续问"能否补充分析诉讼时效方面的抗辩策略？"AI 会在当前模块的上下文中继续深化分析。

**响应式适配：** 平板端缩减为两栏，右侧面板宽度自动调整；手机端变为单栏布局，Tab 横向滑动切换，右侧面板以抽屉方式弹出。

**跨标签页同步：** 若用户在多个浏览器标签页打开同一案件，任何一个标签页中的分析操作（如模块对话生成新结果）都会实时同步到其他标签页，确保数据一致性。

#### 3.2.5 材料处理能力

LexSeek 支持四种类型的案件材料，每种材料有专门的 AI 处理引擎：

| 材料类型 | 处理引擎 | 处理方式 | 输出结果 | 会员限制 |
|---------|---------|---------|---------|---------|
| 文本 | — | 直接存储 | 原始文本 | 所有用户 |
| PDF 文档 | MinerU 文档引擎 | AI 识别文档结构，提取文字、表格、公式 | 结构化 Markdown 文本 | 所有用户 |
| 图片 | OCR 光学识别 | 图片文字识别 | 提取的文字内容 | 所有用户 |
| 音频 | ASR 语音识别 | 智能语音转写为文字 | 转录文本 | 仅专业版/旗舰版 |

材料上传后自动进入处理队列，用户可在材料列表中实时查看处理状态。处理完成的材料会被纳入后续 AI 分析的输入上下文。

### 3.3 AI 案件分析引擎

这是 LexSeek 的核心功能，基于 AI 工作流实现从案情输入到多维度分析输出的全自动化流程。

#### 3.3.1 七大分析模块

LexSeek 提供固定的 7 个分析模块，按以下顺序执行：

| 序号 | 模块名称 | 功能说明 | 用户价值 | 会员权限 |
|------|---------|---------|---------|---------|
| 1 | **生成案件概要** | 基于所有案件材料，自动生成结构化的案件概要，包括事实基础、争议焦点和关键信息 | 5 分钟内掌握案件全貌，省去逐份阅读材料的时间 | 所有等级 |

#### 3.3.2 初始化分析：三阶段模型

初始化分析是 LexSeek 最核心的交互流程，分为三个清晰的阶段：

**阶段一：模块选择**

案件创建完成后，用户自动进入模块选择页面。页面展示 7 个分析模块的勾选列表，默认全部选中。

- 用户可取消勾选不需要的模块（但不可全部取消）
- 若案件此前已做过分析，已完成的模块会标注"已分析"状态，用户可选择仅补充分析未完成的模块，也可勾选重新分析已完成的模块
- 确认选择后点击"开始分析"，进入执行阶段

**阶段二：分析执行**

这是用户感知最强的阶段，AI 实时进行分析并流式输出结果：

- **串行执行机制**：7 个模块按固定顺序依次执行，前一个模块完成后才开始下一个。这确保了后续模块可以利用前序模块的分析结果
- **实时流式输出**：每个模块的分析过程以"逐字呈现"的方式实时展示在界面上，用户不需要等待全部完成就能看到中间结果
- **进度追踪**：页面顶部显示所有已选模块的进度条，每个模块的状态清晰可见：
  - **等待中**（idle）：尚未开始执行
  - **分析中**（streaming）：正在执行，文字逐字输出
  - **已完成**（complete）：分析成功，显示结果摘要
  - **失败**（failed）：分析出错，可单独重试
- **双面板设计**：左侧面板显示当前正在执行的模块的 AI 推理过程；右侧面板同步显示所有已完成模块的结果卡片，形成"实时过程 + 已有结果"的完整视图

**阶段三：分析完成**

当所有选中的模块都执行完毕（无论成功或失败），页面自动进入完成状态：

- 页面底部显示"进入案件详情"按钮，引导用户查看完整结果
- 所有分析结果自动保存，支持版本管理——每次重新分析生成新版本，用户可回溯对比历史版本
- 失败的模块可以单独点击"重试"重新执行，不影响其他模块

**断线恢复：** 若用户在分析过程中关闭页面或网络中断，重新打开页面后系统会自动从断点恢复——已完成的模块结果保留，正在进行的模块继续执行，对用户透明无感知。

#### 3.3.3 中断与恢复机制

在分析过程中，可能出现需要暂停和恢复的场景。LexSeek 设计了完善的中断处理机制：

**场景一：积分不足中断**

当分析消耗积分超过用户余额时，系统自动暂停当前模块并弹出提示弹窗：

- 弹窗显示：当前可用积分数、本次分析需要的积分数、差额
- 用户可选择"立即充值"（跳转到积分/会员购买页面）或"取消"（放弃未完成的模块）
- 充值成功后返回分析页面，系统**自动恢复**被中断的分析，无需用户手动操作
- 已完成的模块结果不受影响

**场景二：用户主动中断**

用户可随时点击"中止分析"按钮主动停止分析流程：

- 当前正在执行的模块会被终止
- 已完成的模块结果全部保留
- 用户后续可随时回到分析页面，选择剩余未完成的模块继续分析

**场景三：网络异常中断**

若因网络问题导致连接断开：

- 系统自动进行重连尝试（最多 3 次）
- 重连过程对用户透明
- 从服务端保存的断点状态恢复，不会丢失已有结果

**场景四：单模块执行失败**

若某个模块因 AI 处理异常而失败：

- 该模块标记为"失败"状态，显示错误信息
- 其余模块不受影响，继续按顺序执行
- 用户可在分析完成后，单独点击失败模块的"重试"按钮进行重新执行

#### 3.3.4 模块对话（深度追问）

初始化分析完成后，用户可以对任意已完成的模块发起"模块对话"，实现二次分析和深度优化。

**使用场景举例：**

- 查看完"抗辩分析"结果后，追问："能否补充分析诉讼时效方面的抗辩策略？"
- 查看完"证据清单"后，追问："针对第 3 项证据，如果没有书面合同，是否有替代证据方案？"
- 对"案情概要"不满意，可要求："请重新生成概要，重点突出合同违约的事实部分"

**交互方式：** 在案件详情页的分析结果区域，每个模块结果下方提供专属输入框，用户输入问题后 AI 在该模块的上下文中继续深化分析。每个模块可创建多个对话会话，历史对话记录可回溯。

#### 3.3.5 思考链推理

用户可以开启"显示思考过程"选项，此时 AI 在输出最终分析结果之前，会先展示其推理过程——包括如何解读案情材料、如何选择法律依据、如何权衡利弊等。这帮助用户理解 AI 分析的逻辑基础，增强对结果的信任度。

### 3.4 法律知识库

LexSeek 内置法律法规数据库，提供两种互补的搜索模式帮助用户快速找到所需法律依据。

#### 3.4.1 全文搜索

**适用场景：** 已知具体的法律名称或关键词，需要精确查找。例如搜索"民法典 519 条"或"租赁合同解除"。

**搜索方式：** 输入关键词后，系统在法律法规的标题和全文中进行精确匹配，返回包含该关键词的所有法律文件。

**搜索结果展示：** 每条结果显示法规名称、颁布机构、发布日期和有效性状态标签（现行有效/已失效/尚未生效）。桌面端以表格形式展示，支持分页浏览（每页 20 条）；移动端以卡片流展示。

#### 3.4.2 法条语义搜索

**适用场景：** 不确定具体的法律条文，需要用自然语言描述问题来查找相关法条。例如输入"房屋租赁纠纷中承租人的权利保障"。

**搜索方式：** AI 理解用户输入的自然语言意图，在法条向量数据库中进行语义匹配，返回与用户意图最相关的法律条文。

**搜索结果展示：** 每条结果显示所属法律名称、法律层级路径（如"第二编 债权编 > 第十章 买卖合同"）、法条内容摘要以及相似度百分比。按相似度从高到低排序。

**相似度评分参考：**

| 相似度范围 | 说明 |
|-----------|------|
| 90% 以上 | 高度相关，通常可直接适用 |
| 70%-90% | 基本相关，需结合案件具体情况判断 |
| 50%-70% | 可能相关，建议仔细核对条文内容 |
| 50% 以下 | 参考价值有限 |

#### 3.4.3 筛选条件

两种搜索模式均支持以下筛选维度：

| 筛选维度 | 可选值 |
|---------|-------|
| 法律类型 | 法律、行政法规、司法解释、规范性文件 |
| 发布机构 | 全国人大、国务院、最高法院、各省市地方机构（动态加载） |
| 有效性状态 | 全部、现行有效、已失效、尚未生效 |
| 发布日期范围 | 起始日期 — 结束日期（仅全文搜索） |

多条件之间为"且"逻辑关系，可组合使用。

#### 3.4.4 法律文本预览

点击搜索结果中的任意法条，弹出详情预览面板：

- 显示法条所属法律的完整名称、编号、发布机构和日期
- 展示法律层级路径（章 > 节 > 条）
- 法条全文以结构化格式呈现
- 支持一键复制法条内容
- 支持前后法条快速导航

### 3.5 法律计算工具集

LexSeek 内置 10 个专业法律计算工具，涵盖律师日常办案中最常见的数值计算场景。所有工具均免费向所有用户开放（部分高级功能限专业版及以上）。

#### 3.5.1 利息计算器

**适用场景：** 民间借贷纠纷、金融借款合同中的利息计算。

**输入参数：**
- 本金金额（元），支持显示中文大写
- 计算方式（四选一）：
  - **自定义利率**：用户手动输入固定利率
  - **LPR 利率**：使用贷款市场报价利率（分 1 年期和 5 年期）
  - **人民银行基准利率**：6 种期限档次（6 个月内、6 月-1 年、1-3 年、3-5 年、5 年以上）
  - **自动分段计算**：以 2019 年 8 月 20 日 LPR 推出为分界点，之前用基准利率，之后用 LPR
- 计息周期：年利率 / 月利率 / 日利率
- 计息起止日期
- 年天数标准：365 天 / 360 天
- 利率调整方式：加点、减点、倍率（如"LPR + 150 基点"）

**计算标准：** 以 2019 年 8 月 20 日为分界点，之前适用人民银行同期贷款基准利率，之后适用 LPR。跨越分界点的计息自动分段计算。利率调整中，1 个基点（BP）= 0.01%。

**输出结果：** 利息总额 + 分段明细表（每段的适用利率、计息天数、利息金额）。支持导出 Excel。

#### 3.5.2 诉讼费用计算器

**适用场景：** 诉讼立案前的费用预算评估。

**支持的案件类型：**
- **财产案件**：按争议金额分档阶梯计算
- **非财产案件**：离婚（固定费用 + 涉及财产部分加收）、人格权、其他非财产
- **知识产权案件**：按费用系数计算
- **劳动争议案件**：固定收费
- **行政案件**：分商标/专利/海事和一般行政

**费用种类：** 包含受理费和申请费两大类。申请费进一步细分为执行费、保全费、支付令费、公示催告费等。

**计算标准：** 依据《诉讼费用交纳办法》中的阶梯费率表，不同争议金额段适用不同费率。

#### 3.5.3 律师费计算器

**适用场景：** 律师费成本评估、案件总预算编制。

**支持六种服务类型：**

| 服务类型 | 关键输入 | 计费特点 |
|---------|---------|---------|
| 民事案件 | 争议金额、复杂度、代理阶段 | 按争议金额百分比，复杂度系数调整 |
| 刑事案件 | 复杂度、预计持续月数、辩护阶段 | 固定收费 + 按时计费 |
| 行政案件 | 案件类型、是否上诉 | 固定收费或争议金额百分比 |
| 法律咨询 | 咨询时长（小时） | 每小时 300-3000 元 |
| 法律文书 | 文书类型、复杂度 | 按类型和复杂度定价 |
| 商事法律服务 | 服务类型、涉及金额 | 固定收费或按项目金额百分比 |

**计算标准：** 依据《律师服务收费管理办法》及各地律师协会发布的指导标准。结果标注"仅供参考，实际费用以律师事务所报价为准"。

#### 3.5.4 延迟履行利息计算器

**适用场景：** 判决后迟延履行的利息计算、债务清偿纠纷。

**计算标准：**

#### 3.5.7 经济补偿/赔偿金计算器

**适用场景：** 劳动争议中的离职赔偿计算、劳动仲裁。

**两种计算模式：**

| 模式 | 适用情形 | 计算公式 |
|------|---------|---------|
| 经济补偿金（N） | 合法解除劳动合同（协商一致、经济性裁员等） | 每满 1 年支付 1 个月工资，不满半年按 0.5 个月 |
| 经济赔偿金（2N） | 违法解除劳动合同 | 经济补偿金的 2 倍 |

**特殊规则：**
- 工资超过当地社会平均工资 3 倍的，按 3 倍计算，且最高补偿年限为 12 年
- 第四十条情形（未提前 30 日通知）需额外支付 1 个月工资
- 内置广州（13193 元）、深圳（14553 元）、珠海（11156 元）等地区社平参考值

**法律依据：**《劳动合同法》第四十七条、第四十八条、第八十七条。

#### 3.5.8 加班费/调休计算器

**适用场景：** 劳动争议中的加班费核算。

**计算标准（《劳动法》第 44 条）：**

| 加班类型 | 工资倍率 | 等价调休比例 |
|---------|---------|-----------|
| 工作日加班 | 不低于 1.5 倍 | 1:1 |
| 休息日加班 | 不低于 2 倍 | 1:1 |
| 法定节假日加班 | 不低于 3 倍 | 1:3 |

输入月基本工资和各类加班小时数后，同时输出加班费金额和等价调休时间。默认月工作日 21.75 天、每天 8 小时。

#### 3.5.9 离婚财产分割计算器

**适用场景：** 离婚协议谈判、离婚诉讼财产分割方案设计。

**计算范围：**
- 共同财产：房产、车辆、存款、投资理财、其他
- 共同债务：房贷、车贷、信用卡、其他

**分割规则：**
- 默认比例：夫妻各 50%（可自定义调整）
- 比例调整因素：一方转移/隐匿财产（可多分 20%-30%）、家暴/出轨过错（无过错方可多分 10%-20%）、子女抚养权归属（抚养方可多分 5%-10%）
- 若涉及子女，可同步计算月抚养费

**输出结果：** 财产概览（总资产/总债务/净资产）、双方各自获得金额、子女抚养费（若涉及）、法律依据参考。

#### 3.5.10 社保追缴计算器

**适用场景：** 社保追缴纠纷、劳动争议仲裁中的欠缴计算。

**六险计算标准：**

| 险种 | 个人缴纳比例 | 单位缴纳比例 |
|------|-----------|-----------|
| 养老保险 | 8% | 16% |
| 医疗保险 | 2% | 8% |
| 失业保险 | 0.5% | 1.5% |
| 工伤保险 | 0% | 0.5% |
| 生育保险 | 0% | 1% |
| 住房公积金 | 7% | 7% |

输入月工资和追缴月数后，分别计算个人和单位应缴部分。各险种比例支持调整为当地实际标准。法律依据为《社会保险法》第七十四条。

#### 3.6.3 会员升级规则

当用户希望从低等级升级到高等级时，LexSeek 提供公平的差价升级机制：

**升级差价计算：**
- 系统根据当前会员的实际支付金额、已使用天数和剩余天数，计算出当前等级的"剩余价值"
- 升级价格 = 目标等级年费 - 当前等级剩余价值

**积分补偿：**
- 升级时，新等级赠送的积分减去旧等级已赠送的积分，差额作为"升级补偿积分"额外赠送
- 旧等级的剩余未使用积分不会丢失，自动转入新等级关联的积分记录

**升级确认：** 系统弹出升级确认弹框，清晰展示当前等级、目标等级、需支付差价和补偿积分金额，用户确认后进入支付流程。

**降级规则：** 不支持降级操作。若需使用低等级，需等待当前会员到期后重新选择续订方案。

#### 3.6.4 积分系统

积分是 LexSeek 的通用消耗货币，用于支付 AI 分析等增值服务的使用成本。

**积分获取途径（10 种来源）：**

| 来源 | 触发条件 | 积分数额 | 说明 |
|------|---------|---------|------|
| 购买会员赠送 | 购买任何等级会员 | 随等级不同（300-12,800） | 购买即赠 |
| 直接购买 | 在积分商城购买积分包 | 按购买数量 | 按需购买 |
| 兑换码赠送 | 输入有效兑换码 | 根据码的配置 | 运营活动发放 |
| 注册赠送 | 新用户首次注册 | 固定数额 | 降低体验门槛 |
| 邀请赠送 | 好友通过邀请链接注册 | 300 积分 | 邀请人和被邀请人均获得 |
| 活动奖励 | 参与官方营销活动 | 活动规则定义 | 不定期举办 |
| 会员升级补偿 | 升级会员等级 | 新旧等级差额 | 公平补偿机制 |
| 会员升级转入 | 升级时保留旧积分 | 旧等级剩余积分 | 用户透明 |
| 积分兑换 | 兑换其他权益 | 负数（消耗） | 如兑换存储空间 |
| 管理员赠送 | 客服补偿等 | 管理员指定 | 特殊场景 |

**积分有效期：** 每笔积分都有生效时间和过期时间。已过期和未生效的积分不计入可用余额。积分管理页面展示可用积分总额、已消耗积分和剩余积分，分别标注"购买积分"和"其他积分"的占比。

**积分消耗机制（预扣-结算两阶段）：**

AI 分析消耗积分采用两阶段安全机制，确保积分扣减的准确性：

1. **预扣阶段**：分析开始前，系统检查用户可用积分是否充足，若充足则"预扣"相应积分（标记为待结算状态），保证分析过程中积分不被其他操作消耗
2. **结算阶段**：分析完成后，根据实际消耗量（如 AI 模型调用的 token 数）计算最终积分消耗，将预扣记录更新为已结算状态
3. **回滚机制**：若分析中途失败，预扣的积分会自动回滚，返还给用户

**积分不足时的处理：** 当用户积分不足以支付当前分析操作时，AI 分析流程自动中断，弹出提示弹窗显示"您需要 X 积分来完成本次分析，当前仅有 Y 积分"。用户可选择"立即充值"跳转到购买页面，充值成功后返回分析页面系统自动恢复被中断的分析。

## 五、商业模式

### 5.1 收入模型

LexSeek 采用"会员订阅 + 积分消耗"的复合收入模型：

| 收入来源 | 说明 | 特点 |
|---------|------|------|
| **会员订阅收入** | 基础版/专业版/旗舰版年费 | 稳定的经常性收入 |
| **积分销售收入** | 单独购买积分包 | 按需付费，灵活补充 |
| **引流型产品** | ¥9.9 新手旗舰套餐（限购 1 次） | 低门槛转化，培养付费习惯 |

### 5.2 定价策略

- **阶梯式权益差异化**：不同价位提供明确不同的功能范围，引导用户根据需求选择
- **年付优惠引导**：年付价格相比按月折算有显著折扣（如基础版年付 ¥365 vs 理论月付更高），鼓励长期订阅
- **新手限购低价策略**：¥9.9 即可体验 1 个月旗舰版全部功能（含 300 积分），降低首次付费心理门槛，每人限购 1 次确保引流目的
- **积分消耗制**：分析按实际使用消耗积分，确保用户按需付费而非为未使用的功能买单

### 5.3 增长策略

1. **降低体验门槛**：注册即赠 7 天旗舰版试用；首页示例案件无需注册即可体验完整分析流程
2. **邀请裂变**：每邀请一位好友注册，双方各获 300 积分，形成用户自传播
3. **兑换码定向推广**：通过兑换码与 KOL、律所协会等合作，精准获客
4. **限时活动横幅**：Dashboard 首页展示限时活动信息，利用紧迫感驱动转化
5. **防滥用机制**：禁止转卖出租，单用户并发分析限制为 1，保障正常使用环境

### 5.4 关键业务指标

| 指标 | 定义 |
|------|------|
| 注册转化率 | 网站访客 → 完成注册的用户比例 |
| 付费转化率 | 注册用户 → 首次付费用户比例 |
| ARPU | 每付费用户平均收入 |
