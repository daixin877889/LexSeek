# 案件分析功能文档

## 概述

本文档整理了 `lexseek_web`（前端）和 `lexseekApi`（后端）两个项目中案件分析功能的业务逻辑和技术细节。系统最终采用 **Socket.io** 实现实时通信，支持流式输出分析结果。

---

## 一、系统架构

### 1.1 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue 3 + Pinia + Socket.io-client |
| 后端 | Express + Socket.io + LangGraph |
| AI 引擎 | LangChain + LangGraph |
| 数据库 | PostgreSQL + Prisma ORM |
| 向量数据库 | PGVector |

## 二、Socket 事件定义

### 2.1 客户端 → 服务端事件

| 事件名 | 描述 | 参数 |
|--------|------|------|
| `generateCaseSessionId` | 生成案件会话 ID | 无 |
| `createCase` | 创建案件 | `CreateCaseEventData` |
| `file:material:conversion:create` | 材料转换（文档/图片/音频） | `MaterialConversionData` |
| `file:material:conversion:query` | 查询材料转换状态 | `FileMaterialConversionQueryData` |
| `case:userConfirm` | 用户确认案件基本信息 | `CaseUserConfirmData` |
| `case:session:analysis:record` | 获取案件分析记录 | `{ sessionId }` |
| `case:session:message` | 获取会话所有消息 | `{ sessionId }` |
| `case:session:user:available:modules` | 获取用户可用分析模块 | 无 |
| `case:analysis:startTask` | 开始分析任务 | `CaseAnalysisStartTaskData` |
| `case:session:info` | 根据 sessionId 获取案件信息 | `{ sessionId }` |

### 2.2 服务端 → 客户端事件

| 事件名 | 描述 | 数据结构 |
|--------|------|----------|
| `case:created` | 案件创建成功 | `{ caseId, sessionId }` |
| `case:analysis:message` | 分析过程消息（流式） | `LanggraphResult` |
| `error` | 错误消息 | `{ event, message }` |

### 3.2 案件材料类型

```typescript
enum CaseMaterialType {
  CASE_CONTENT = 1,  // 文本内容

interface CaseMaterialParamType {
  type: CaseMaterialType;
  name?: string;
  content?: string;      // 文本内容（type=1 时必填）
```

```typescript
enum LanggraphResultType {
```

## 九、积分消耗机制

### 9.1 积分扣减流程

1. 用户选择分析模块
2. 前端计算总积分（考虑折扣）
3. 后端验证用户积分余额
4. 执行分析任务
5. 任务完成后扣减积分
6. 记录积分消耗日志

### 9.2 积分计算

```typescript
// 计算实际扣减积分
```

## 十、错误处理

### 10.1 常见错误类型

| 错误 | 描述 | 处理方式 |
|------|------|----------|
| 用户未认证 | token 无效或过期 | 重新登录 |
| 案件不存在 | caseId 无效 | 返回错误提示 |
| 无权限 | 用户无权访问该案件 | 返回权限错误 |
| 积分不足 | 用户积分余额不足 | 提示充值 |
| 模块无权限 | 用户会员等级不足 | 提示升级会员 |
| 材料转换失败 | 文档/图片/音频识别失败 | 重试或手动输入 |

### 10.2 错误回调格式

```typescript
callback?.({ 
  success: false, 
  error: errorMessage 
});
```

## 十三、总结

案件分析功能采用 Socket.io 实现实时通信，结合 LangGraph 构建 AI 工作流，支持：

1. **多类型材料处理**：文本、文档、图片、音频
2. **流式输出**：实时展示分析过程和结果
3. **模块化分析**：支持多种分析模块和文书生成
4. **版本管理**：支持分析结果的历史版本
5. **重新生成**：支持基于用户反馈重新生成
6. **积分系统**：按模块计费，支持会员折扣

迁移时需要注意保持 Socket 事件的兼容性，以及 LangGraph 工作流的正确配置。
